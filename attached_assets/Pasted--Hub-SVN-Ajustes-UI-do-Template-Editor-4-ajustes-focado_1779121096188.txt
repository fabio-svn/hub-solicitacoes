# Hub SVN — Ajustes UI do Template Editor

4 ajustes focados em aproveitar melhor o espaço e simplificar.

---

## #1 — Remover sidebar global no editor

O editor de templates é uma ferramenta intensa em uso de tela. Quando aberto, ele deve ocupar a área inteira — sem a sidebar de admin global.

**Implementação:**

A página de edição de template (`admin-templates.html` ou similar, no modo "editing") deve **não renderizar a sidebar do Shell**, mesmo que o usuário seja admin.

Adicionar opção no `Shell.render`:

```js
Shell.render({
  activeRoute: 'admin-templates',
  contentEl: document.getElementById('pageContent'),
  hideSidebar: true,  // ← novo
});
```

Quando `hideSidebar: true`, o shell renderiza só `header + main` (mesmo pra admin). Equivale ao layout do usuário não-admin.

No CSS, adicionar variação:
```css
.app-shell.no-sidebar {
  grid-template-areas:
    "header"
    "main";
  grid-template-columns: 1fr;
}
.app-shell.no-sidebar .app-sidebar { display: none; }
```

E o `Shell.render` aplica `.no-sidebar` ao `.app-shell` quando `hideSidebar=true`.

**Onde aplicar:** apenas na view de **edição** de template. A view de listagem `/admin/templates` continua com sidebar normal.

## #2 — Remover header no editor, botão Voltar acima das layers

Junto com a sidebar, o **header também sai** quando editor está aberto. O editor vira full-screen mesmo.

O botão "Voltar" deixa de viver no header — passa a ficar **no topo do painel esquerdo das layers**:

```
┌─────────────────────────────────────────────────────┐
│ [← Voltar]                                          │  ← botão acima das layers
│ ───────────────────                                 │
│ Layers                                              │
│ [T] Mensagem                                        │
│ [T] Nome do cliente                                 │
│ [🖼] Logo                                            │
│                                                     │
│ [+ Adicionar layer]                                 │
├─────────────────────────────────────────────────────┤
│           [canvas escuro com o artboard]            │
└─────────────────────────────────────────────────────┘
```

**Implementação:**

Mudar `Shell.render` (ou criar `Shell.renderEditor`) pra aceitar:

```js
Shell.render({
  hideSidebar: true,
  hideHeader: true,  // ← novo
  contentEl: ...
});
```

Quando `hideHeader=true`, não renderiza o `.app-header`. CSS:

```css
.app-shell.no-header {
  grid-template-areas: "main";
  grid-template-rows: 1fr;
}
.app-shell.no-header .app-header { display: none; }
```

Combinado com `no-sidebar`, o editor ocupa 100% da viewport.

**Botão Voltar no painel de layers:**

No HTML do editor, no topo do painel esquerdo:

```html
<div class="editor-sidebar">
  <button onclick="handleVoltar()" class="editor-back-btn">← Voltar</button>
  <div class="editor-separator"></div>
  
  <h3 class="editor-section-title">Layers</h3>
  <div class="layers-list">...</div>
  
  <button class="add-layer-btn">+ Adicionar layer</button>
</div>
```

CSS:
```css
.editor-back-btn {
  background: transparent;
  border: 1px solid #e5e3df;
  padding: 8px 14px;
  border-radius: 6px;
  font-size: 0.85rem;
  cursor: pointer;
  width: 100%;
  text-align: left;
  color: #221b19;
  margin-bottom: 12px;
}
.editor-back-btn:hover { background: rgba(0,0,0,0.04); }

.editor-separator {
  border-bottom: 1px solid #e5e3df;
  margin-bottom: 12px;
}
```

Manter a lógica do `handleVoltar` igual (modal de unsaved changes etc.).

## #3 — Remover "Texto (linha)" — só manter "Texto (área)"

A diferença entre `text-line` e `text-block` era artificial. O `text-block` (texto área) já suporta texto de uma linha — basta o usuário definir altura pequena e uma linha de conteúdo. Manter dois tipos confunde.

**Implementação:**

1. **No botão "Adicionar layer"** do editor, remover a opção "Texto (linha)". Manter apenas:
   - "Texto" (era "Texto (área)") — renomear pra simplesmente "Texto"
   - "Imagem"

2. **Renderização**: layers existentes do tipo `text-line` continuam sendo renderizadas pelo renderer (compatibilidade). Não migrar dados, deixar como estão.

3. **No editor**, layers existentes de tipo `text-line` aparecem na lista normalmente, com ícone de texto, e podem ser editadas (mesmo painel de propriedades do text-block).

4. **Renomear nos arquivos**: `text-block` continua sendo o tipo interno (não muda no schema), mas na UI a label fica só "Texto":

   ```js
   const LAYER_LABELS = {
     'text-block': 'Texto',
     'text-line':  'Texto',  // legacy, mesmo label
     'image':      'Imagem',
   };
   ```

5. **Ícone único pra texto:** unifica os ícones de texto-linha (T) e texto-área (Tt) — usa o ícone mais expressivo (o "Tt" ou um ícone de parágrafo) pra todas as layers de texto.

## #4 — Botão de upload/library no campo de bg do Documento

Na aba "Documento" das propriedades do template (onde fica o campo de URL do background), o input atualmente é só texto. Outras layers (image) já têm botões `⬆ Upload` e `📁 Library` ao lado do input — replicar aqui.

**Implementação:**

No painel de propriedades quando "Documento" está selecionado, o campo do bg deve ser:

```html
<div class="field-with-asset-btns">
  <label>Background</label>
  <div class="asset-input-row">
    <input type="text" id="bgUrl" value="${currentTemplate.bg.url}" 
           onchange="updateBgUrl(this.value)" />
    <button onclick="uploadAssetForBg()" class="asset-btn upload-btn" title="Fazer upload">
      ⬆
    </button>
    <button onclick="openAssetLibraryForBg()" class="asset-btn library-btn" title="Escolher da biblioteca">
      📁
    </button>
  </div>
</div>
```

```js
function uploadAssetForBg() {
  // Mesmo handler que já existe pra layers de imagem,
  // mas o callback de "URL pronto" atualiza o bg em vez do source da layer
  triggerFileUpload((url) => {
    currentTemplate.bg = { type: 'static', url };
    document.getElementById('bgUrl').value = url;
    pushHistory();
    renderCanvas();
    markDirty();
  });
}

function openAssetLibraryForBg() {
  // Mesma library, callback diferente
  openAssetLibrary((url) => {
    currentTemplate.bg = { type: 'static', url };
    document.getElementById('bgUrl').value = url;
    pushHistory();
    renderCanvas();
    markDirty();
  });
}
```

A ideia é reaproveitar as funções existentes de upload e library (que já funcionam pras layers de imagem) com callbacks customizados.

CSS do row (se ainda não existe):

```css
.asset-input-row {
  display: flex;
  gap: 6px;
  align-items: center;
}
.asset-input-row input { flex: 1; }
.asset-btn {
  background: rgba(0,0,0,0.05);
  border: none;
  padding: 6px 10px;
  border-radius: 4px;
  cursor: pointer;
  font-size: 1rem;
}
.asset-btn:hover { background: rgba(0,0,0,0.1); }
```
