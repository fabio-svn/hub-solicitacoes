Ah, então é uma funcionalidade que faltou — reordenar layers. Não é bug, é feature missing.
Manda pro agente:

No editor de templates, a lista de layers no painel lateral mostra as camadas mas não permite reordenar. Isso é crítico: a ordem na lista define a ordem de renderização (z-index), então sem reordenar é impossível trazer uma layer pra cima quando ela foi criada embaixo de outra.
Adicionar 2 mecanismos de reordenação:
1. Botões de seta em cada item da lista
No item de cada layer, adicionar dois botões pequenos à direita (ao lado dos ícones de visibilidade/lock):
html<div class="layer-item">
  <span class="layer-icon">T</span>
  <span class="layer-name">Nome da camada</span>
  <button class="layer-up"   onclick="moveLayerUp('${id}')"   title="Mover pra cima">↑</button>
  <button class="layer-down" onclick="moveLayerDown('${id}')" title="Mover pra baixo">↓</button>
  <button class="layer-visibility">👁</button>
  <button class="layer-lock">🔓</button>
</div>
Funções:
jsfunction moveLayerUp(id) {
  const idx = currentTemplate.layers.findIndex(l => l.id === id);
  if (idx <= 0) return;  // já no topo (ou não encontrada)
  const [layer] = currentTemplate.layers.splice(idx, 1);
  currentTemplate.layers.splice(idx - 1, 0, layer);
  pushHistory();
  renderLayerList();
  renderCanvas();
  markDirty();
}

function moveLayerDown(id) {
  const idx = currentTemplate.layers.findIndex(l => l.id === id);
  if (idx === -1 || idx >= currentTemplate.layers.length - 1) return;
  const [layer] = currentTemplate.layers.splice(idx, 1);
  currentTemplate.layers.splice(idx + 1, 0, layer);
  pushHistory();
  renderLayerList();
  renderCanvas();
  markDirty();
}
Desabilitar visualmente o ↑ quando a layer é a primeira da lista, e o ↓ quando é a última.
2. Drag-and-drop na lista (mais polido)
Tornar os itens da lista arrastáveis pra reordenar via drag. Usar HTML5 drag API ou interact.js (já é dependência do projeto). Implementação básica com interact.js:
jsinteract('.layer-item').draggable({
  listeners: {
    start(event) {
      event.target.classList.add('dragging');
    },
    move(event) {
      // Visual feedback (deslocamento Y)
      const target = event.target;
      const y = (parseFloat(target.getAttribute('data-y')) || 0) + event.dy;
      target.style.transform = `translateY(${y}px)`;
      target.setAttribute('data-y', y);
    },
    end(event) {
      event.target.classList.remove('dragging');
      event.target.style.transform = '';
      event.target.removeAttribute('data-y');
      
      // Calcular nova posição baseada em onde foi solto
      const draggedId = event.target.dataset.layerId;
      const items = Array.from(document.querySelectorAll('.layer-item'));
      const dropY = event.client.y;
      
      let newIdx = items.findIndex(item => {
        const rect = item.getBoundingClientRect();
        return dropY < rect.top + rect.height / 2;
      });
      if (newIdx === -1) newIdx = items.length - 1;
      
      reorderLayer(draggedId, newIdx);
    }
  }
});

function reorderLayer(id, newIdx) {
  const oldIdx = currentTemplate.layers.findIndex(l => l.id === id);
  if (oldIdx === -1 || oldIdx === newIdx) return;
  const [layer] = currentTemplate.layers.splice(oldIdx, 1);
  currentTemplate.layers.splice(newIdx, 0, layer);
  pushHistory();
  renderLayerList();
  renderCanvas();
  markDirty();
}
Adicionar um "handle" visual (ícone ⋮⋮ ou ≡) no início de cada item pra indicar que pode arrastar.
3. Z-index respeitando a ordem
Confirmar que ao renderizar as layers no canvas, o z-index é atribuído conforme a ordem no array:
jsconfig.layers.forEach((layer, idx) => {
  const el = createLayerEl(layer);
  el.style.zIndex = String(idx + 1);
});
Layers no fim do array (índices maiores) ficam visualmente em cima.
4. Convenção da ordem
Definir e documentar (comentário no código):

Topo da lista (índice 0) = layer ATRÁS no canvas (fundo)
Fim da lista (último índice) = layer NA FRENTE no canvas (topo)

Ou inverter, conforme preferência. O importante é consistência: o que aparece em cima na lista é o que está em cima no canvas (ou vice-versa, mas claramente definido).
Convenção mais comum em editores (Figma, Photoshop): topo da lista = na frente. Pra adotar essa, o forEach precisa atribuir z-index reverso:
jsconst total = config.layers.length;
config.layers.forEach((layer, idx) => {
  el.style.zIndex = String(total - idx);
});
E moveLayerUp na lógica deve mover pra índice menor (que visualmente fica em cima da lista).
Escolher uma convenção e aplicar consistentemente em: renderização do canvas, função de reordenar, e botões ↑/↓.
Validação: criar template com layer image grande + layer text. Texto fica atrás. Clicar ↑ na lista → texto vai pra cima e fica visível por cima da imagem no canvas. Arrastar texto pra outra posição na lista via drag-and-drop → ordem muda no canvas em tempo real.
