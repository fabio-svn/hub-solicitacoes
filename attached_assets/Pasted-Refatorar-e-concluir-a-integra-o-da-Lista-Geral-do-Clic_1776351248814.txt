Refatorar e concluir a integração da **Lista Geral do ClickUp** para os formulários não-eventos do hub.

## Objetivo

Aplicar integração completa para os tipos:

* artes-divulgacao
* atualizacao-material
* conteudo-pdf-informativo
* conteudo-pdf-ebook
* apresentacao-nova
* apresentacao-atualizar
* pagina-assessores-dados
* pagina-assessores-atualizacao

A integração deve:

1. continuar criando task normalmente
2. preencher custom fields da Lista Geral
3. gerar título humanizado padronizado
4. gerar descrição limpa e útil (sem JSON bruto)
5. usar **Setor** do usuário autenticado via Microsoft no título e descrição

---

# PARTE 1 — Novo título das tasks

## Formato obrigatório

```text
[Tipo de material] (Setor) - Título
```

## Exemplos

* `[Arte de Divulgação] (Comercial) - Evento XP Maio`
* `[PDF Ebook] (Marketing) - Guia Previdência 2026`
* `[Apresentação Nova] (Private Banking) - Wealth Planning Summit`
* `[Atualização de Material] (Operações) - Deck Institucional`
* `[Página de Assessores] (Expansão) - João Sardeto`

## Humanização obrigatória do tipo

Converter:

* artes-divulgacao → Arte de Divulgação
* conteudo-pdf-informativo → PDF Informativo
* conteudo-pdf-ebook → PDF Ebook
* apresentacao-nova → Apresentação Nova
* apresentacao-atualizar → Atualização de Apresentação
* atualizacao-material → Atualização de Material
* pagina-assessores-dados → Página de Assessores
* pagina-assessores-atualizacao → Página de Assessores

## Fonte do Setor

Usar dados da integração Microsoft / usuário autenticado.

Prioridade:

1. setor já existente no objeto user/session
2. department Microsoft
3. base interna existente
4. fallback: `(Geral)`

## Fallbacks

Se não houver título:

```text
[Arte de Divulgação] (Marketing)
```

Se não houver setor:

```text
[Arte de Divulgação] - Evento XP Maio
```

# PARTE 3 — Regras de mapeamento por tipo

## 1. Artes de Divulgação

Tipo = Arte de Divulgação

Mapear:

* nome → Nome do solicitante
* tipo técnico → Tipo de solicitação
* subtipo se existir
* titulo → Título
* finalidade → Finalidade
* publicoAlvo → Público-alvo
* prazoEntrega → Prazo de entrega
* canais → Canais
* arquivoApoio → Arquivo de apoio

Descrição deve conter:

* conteúdo / briefing
* canal personalizado
* observações

# PARTE 4 — Descrição humanizada

## Regra principal

NÃO usar mais:

* JSON.stringify(dados)
* dump bruto do payload

## Estrutura base

```text
👤 Solicitante
• Nome: ...
• Setor: ...

📌 Resumo
• Tipo: ...
• Finalidade: ...
• Prazo: ...

📝 Detalhes
...

📎 Arquivos
...
```

Mostrar apenas campos preenchidos.

Não mostrar:

* null
* undefined
* vazio
* placeholders
