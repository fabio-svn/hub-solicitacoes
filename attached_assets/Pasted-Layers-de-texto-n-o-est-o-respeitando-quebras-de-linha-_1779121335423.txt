Layers de texto não estão respeitando quebras de linha. Quando o usuário insere texto com \n (Enter no input ou no campo do form), o renderer ignora e quebra apenas por largura.
Investigar e corrigir nas 3 camadas:
1. Input do editor (admin-templates.html)
O campo onde se digita o conteúdo de uma layer de texto deve ser <textarea> (não <input>), pra permitir múltiplas linhas. Confirmar que:
html<textarea 
  id="layerContent" 
  rows="4"
  onchange="updateLayerContent(this.value)"
  placeholder="Conteúdo do texto. Use Enter para quebrar linha."
></textarea>
Ao salvar, o valor do textarea preserva \n no JSON do config (verificar no DB depois do save).
2. Renderer de texto (template-renderer.ts)
A função que renderiza texto deve:

Primeiro splitar por \n — cada pedaço é uma "linha forçada" do usuário
Depois aplicar wrap por largura em cada pedaço — se uma linha do usuário exceder a largura da box, quebra adicional
Linhas vazias (resultado de \n\n no input) viram espaço vazio com altura = lineHeight no rendering

Pseudo-código:
tsfunction renderTextBlock(text, layer) {
  const userLines = text.split('\n');           // respeita quebras manuais
  const renderedLines = [];
  
  for (const userLine of userLines) {
    if (userLine === '') {
      // Linha vazia — empurra como blank spacer
      renderedLines.push('');
      continue;
    }
    // Wrap por largura
    const wrapped = wrapTextByWidth(userLine, layer.width, layer.fontSize, font);
    renderedLines.push(...wrapped);
  }
  
  // Desenhar cada linha com altura = lineHeight, posições incrementais
  let y = layer.y;
  for (const line of renderedLines) {
    drawText(line, layer.x, y, ...);
    y += layer.lineHeight || layer.fontSize * 1.2;
  }
}
3. Preview no editor
O preview no canvas também deve usar a mesma lógica — splitar por \n antes de wrap. Confirmar que renderCanvas (live preview) replica o comportamento do renderer server-side.
4. Placeholders vindos de form submission
Quando o texto da layer é placeholder ({{mensagem}}), o valor vem do form. O form provavelmente usa <textarea> pro campo de mensagem (que já permite quebras). Confirmar que:

O payload enviado preserva \n (não escapa, não converte pra espaço)
O backend não normaliza/trim que destrói quebras
Ao substituir {{mensagem}} no template, o texto resultante mantém \n

Validação

Criar template com layer de texto. No campo de conteúdo, digitar:
Linha 1

Linha 3 depois de uma linha vazia
Linha 4 junto

Salvar e verificar preview no editor → deve mostrar 4 linhas visíveis com gap no meio
Submeter solicitação (se for placeholder) com texto multi-linha → arte gerada deve preservar quebras
Layer com texto longo que extrapola a largura E tem quebras manuais → ambos os tipos de quebra coexistem (manuais primeiro, wrap por largura em cada pedaço)
