Bug 1: POST /api/admin/art-templates está recebendo config: null do frontend e crashando.
Olhando a network: payload enviado é
json{ "tipo": "cartao-comemorativo", "name": "Cartão comemorativo", "config": null }
O frontend, ao criar template "em branco", precisa gerar uma config padrão em vez de enviar null. Em admin-templates.html, na função que monta o payload do "Criar":
jsasync function createNewTemplate(tipo, name, startFromBlank, sourceTemplateId) {
  let config;
  if (startFromBlank) {
    config = {
      tipo,
      canvas: { width: 1080, height: 1080 },
      bg: { type: 'static', url: '' },
      layers: [],
    };
  } else {
    // Clonar de existente
    const source = await fetch(`/api/admin/art-templates/${sourceTemplateId}`).then(r => r.json());
    config = JSON.parse(JSON.stringify(source.config));
    config.tipo = tipo;  // garantir que tipo é o novo, não o do clone
  }

  const res = await fetch('/api/admin/art-templates', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    credentials: 'include',
    body: JSON.stringify({ tipo, name, config }),
  });
  // ...
}

Bug 2: EISDIR no preview quando URL é vazia.
Em src/services/template-renderer.ts, função getRemoteAsset (e/ou no ponto onde o bgBuf é carregado), proteger contra URLs vazias/inválidas:
tsasync function getRemoteAsset(url: string): Promise<Buffer> {
  if (!url || url.trim() === '') {
    throw new Error('URL de asset vazia');
  }
  if (url.startsWith('http')) {
    const res = await fetch(url);
    if (!res.ok) throw new Error(`Falha ao buscar ${url}: ${res.status}`);
    return Buffer.from(await res.arrayBuffer());
  }
  // path local
  const fullPath = url.startsWith('/') ? url : path.join(ASSETS_DIR, url);
  const stats = await fs.promises.stat(fullPath).catch(() => null);
  if (!stats) throw new Error(`Asset não encontrado: ${fullPath}`);
  if (stats.isDirectory()) throw new Error(`Path é diretório, não arquivo: ${fullPath}`);
  return fs.promises.readFile(fullPath);
}
E em renderFromTemplate, quando o template é novo/em branco (sem bg url ainda), em vez de crashar, renderizar um placeholder cinza com texto "Sem background":
tslet bgBuf: Buffer;
try {
  bgBuf = await getRemoteAsset(bgUrl);
} catch (err) {
  console.warn(`[render] bg inválido (${err.message}), usando placeholder`);
  // Gera um bg cinza no tamanho do canvas como placeholder
  bgBuf = await sharp({
    create: {
      width: template.canvas.width,
      height: template.canvas.height,
      channels: 4,
      background: { r: 64, g: 64, b: 68, alpha: 1 },
    },
  }).png().toBuffer();
}
