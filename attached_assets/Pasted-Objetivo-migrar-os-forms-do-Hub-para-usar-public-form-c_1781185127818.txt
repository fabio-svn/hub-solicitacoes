public/form-core.js expõe window.FormCore:
- FormCore.initForm({ tipo, redirectPath?, draft?, onReady? }) — faz _configReady, Auth.init(), gate de login, Shell.render(), popula #setor com SETORES, aplica perfil (Auth.aplicarPerfilNoCampo nos campos padrão) e chama onReady(). Com draft:true, salva/restaura rascunho em sessionStorage['form-'+tipo] automaticamente.
- FormCore.validateRequired(extraValidate?) — valida [required], marca .field-invalid + .field-error, rola pro primeiro inválido, retorna boolean. extraValidate(invalidFields) é opcional pra regras extras.
- FormCore.markInvalid(invalidFields, containerEl, msg) — helper pras regras extras.
- FormCore.submit({ tipo, dados, files? }) — trava #btnSubmit, monta FormData (tipo_solicitacao + dados JSON + anexos), POST /api/solicitacoes; no ok grava svn_ultimo_resumo e redireciona pra /thankyou.html; no erro mostra #submitError e destrava. files = array de ids de input file (ex.: ['arquivoApoio']).
