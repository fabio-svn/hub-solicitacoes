## Fix R3.2 — Try/catch no submit físico do cartão de visita

Em form-cartao-visita.html, função submitFormFisico (~linha 290-325):

LOCALIZAR o submit que NÃO tem try/catch (físico). 

ENVOLVER toda a parte `const res = await fetch(...)` até o final do 
handling de erro num try/catch, seguindo o padrão JÁ USADO em 
submitFormDigital no mesmo arquivo (~linha 338+):

```js
try {
  const res = await fetch('/api/solicitacoes', { method: 'POST', body: formData });
  const d = await res.json().catch(() => ({}));
  if (res.ok) {
    // ... lógica de sucesso já existente ...
  } else {
    throw new Error(d.error || 'submit_failed');
  }
} catch (e) {
  const errEl = document.getElementById('submitErrorFisico');
  if (errEl) { errEl.style.display = 'block'; errEl.textContent = 'Erro ao enviar. Tente novamente ou verifique sua conexão.'; }
  if (window.formUnlockUI) window.formUnlockUI(btn); else { btn.disabled = false; btn.textContent = 'Enviar'; }
}
```

PADRONIZAR também o submit do físico pra usar `throw new Error(d.error)` 
ao invés de inline handling no else (acompanha o pattern do digital).
