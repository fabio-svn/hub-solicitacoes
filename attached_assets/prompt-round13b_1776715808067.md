## 3. Mostrar apenas status em uso no gráfico "Por status"

### Root cause: `adminGraficoData.porStatus` retorna TODOS os status já usados
### historicamente, mas ao filtrar o período pode trazer status com count > 0
### que não estão sendo usados agora. Não é esse o problema —
### o problema é que ao inicializar `graficoFiltrosAtivos` com todos os IDs,
### status com count = 0 não aparecem (o backend só retorna status com count > 0).
### O problema REAL: ao trocar de "Por tipo" para "Por status", o Set de filtros
### é resetado para null e reinicializado com TODOS os itens de porStatus.
### Se porStatus tem 8 entradas, todas aparecem mesmo que visualmente o gráfico
### seja pequeno. Isso é correto — cada entrada tem count > 0.
### A leitura do usuário é que status "irrelevantes" aparecem no dropdown.
### Solução: filtrar itens com count === 0 (não devem existir, mas por segurança)
### E mais importante: ordenar por count DESC para que os mais usados apareçam primeiro.

```js
// Em renderAdminGrafico(), na montagem de todosItens para status:
// Substituir:
const todosItens = ... adminGraficoData.porStatus.map(i => ({
  id: i.status,
  label: STATUS_SOLICITACAO?.find(s => s.id === i.status)?.label || i.status,
  value: i.count,
  cor: STATUS_SOLICITACAO?.find(s => s.id === i.status)?.bg || null,
}));

// Por (filtrando zeros e ordenando por count DESC):
const todosItens = adminGraficoData.porStatus
  .filter(i => i.count > 0) // garantia — backend não deve enviar zeros
  .sort((a, b) => b.count - a.count)
  .map(i => ({
    id: i.status,
    label: STATUS_SOLICITACAO?.find(s => s.id === i.status)?.label || i.status,
    value: i.count,
    cor: STATUS_SOLICITACAO?.find(s => s.id === i.status)?.bg || null,
  }));

// Fazer o mesmo para porTipo:
const todosItens = adminGraficoData.porTipo
  .filter(i => i.count > 0)
  .sort((a, b) => b.count - a.count)
  .map(i => ({
    id: i.tipo,
    label: (typeof TIPO_SOLICITACAO_LABELS !== 'undefined' && TIPO_SOLICITACAO_LABELS[i.tipo]) || i.tipo,
    value: i.count,
  }));
```

### 4b. Animação de entrada nas fatias do canvas (efeito de crescimento)

```js
// Substituir o desenho estático das fatias por animação de crescimento:

// Após o fade out (dentro do setTimeout), substituir o bloco de desenho por:

const total = itens.reduce((a, b) => a + b.value, 0);
if (total === 0) {
  // estado vazio — igual ao existente
  return;
}

const CORES = ['#AC3631','#221B19','#4a7c2f','#1e40af','#9333ea','#ea580c','#0891b2','#be123c','#854d0e','#166534'];
canvas.width = 200; canvas.height = 200;
const ctx = canvas.getContext('2d');
const cx = 100, cy = 100, r = 85, rInner = 50;

// Ângulos alvo para cada fatia
const slices = itens.map((item, i) => ({
  angle: (item.value / total) * 2 * Math.PI,
  cor: item.cor || CORES[i % CORES.length],
}));

// Animação de crescimento (expandir de 0 a 1 em 350ms)
const duration = 350;
const startTime = performance.now();

function drawFrame(now) {
  const elapsed = now - startTime;
  const progress = Math.min(elapsed / duration, 1);
  // easeOutCubic
  const ease = 1 - Math.pow(1 - progress, 3);

  ctx.clearRect(0, 0, 200, 200);

  let startAngle = -Math.PI / 2;
  slices.forEach(slice => {
    const currentAngle = slice.angle * ease;
    ctx.beginPath();
    ctx.moveTo(cx, cy);
    ctx.arc(cx, cy, r, startAngle, startAngle + currentAngle);
    ctx.closePath();
    ctx.fillStyle = slice.cor;
    ctx.fill();
    startAngle += currentAngle;
  });

  // Buraco central
  ctx.beginPath();
  ctx.arc(cx, cy, rInner, 0, 2 * Math.PI);
  ctx.fillStyle = getComputedStyle(document.body).getPropertyValue('--paper-white').trim() || '#FFF8F3';
  ctx.fill();

  // Número central (aparece após 60% da animação)
  if (progress > 0.6) {
    const alpha = Math.min((progress - 0.6) / 0.4, 1);
    ctx.globalAlpha = alpha;
    ctx.fillStyle = '#221B19';
    ctx.font = 'bold 24px "Nunito Sans", sans-serif';
    ctx.textAlign = 'center';
    ctx.textBaseline = 'middle';
    ctx.fillText(total, cx, cy - 6);
    ctx.font = '11px "Nunito Sans", sans-serif';
    ctx.fillStyle = 'rgba(34,27,25,0.4)';
    ctx.fillText('total', cx, cy + 14);
    ctx.globalAlpha = 1;
  }

  if (progress < 1) {
    requestAnimationFrame(drawFrame);
  }
}

requestAnimationFrame(drawFrame);

// Legenda aparece junto com o fade in do canvas (já controlado pelo opacity)
legenda.innerHTML = itens.map((item, i) => {
  const pct = Math.round((item.value / total) * 100);
  const cor = item.cor || CORES[i % CORES.length];
  return '<div style="display:flex;align-items:center;gap:8px">' +
    '<div style="width:10px;height:10px;border-radius:2px;background:' + cor + ';flex-shrink:0"></div>' +
    '<span style="font-size:0.8rem;flex:1;line-height:1.3">' + esc(item.label) + '</span>' +
    '<span style="font-size:0.8rem;font-weight:700;opacity:0.7">' + pct + '%</span>' +
    '<span style="font-size:0.75rem;opacity:0.4">' + item.value + '</span>' +
    '</div>';
}).join('');

// Fade in da legenda junto com o canvas
canvas.style.opacity = '1';
legenda.style.opacity = '1';
```

## OBSERVAÇÕES

- Apenas `dashboard.html` — nenhum build necessário
- A animação usa `requestAnimationFrame` nativo — sem dependências externas
- O easing `easeOutCubic` dá sensação de desaceleração natural ao final
- O número central aparece apenas após 60% da animação para não
  conflitar visualmente com o crescimento das fatias
- Ordenar por count DESC no dropdown garante que os itens mais
  relevantes ficam no topo, reduzindo a percepção de "ruído"
