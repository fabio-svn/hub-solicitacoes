# Hub SVN — Página individual de solicitação de automação (preview + download)

## Contexto e objetivo

Hoje a página individual de cada solicitação (ao clicar num item da lista "Minhas solicitações") provavelmente mostra um resumo detalhado dos campos do form, status, comentários, etc. Pra solicitações de automação — onde o entregável principal é o material gerado — esse detalhamento é ruído. O usuário quer ver o material e baixar, ponto.

A página de **assinaturas de e-mail** já tem esse padrão (preview + download, clean e direto). Replicar essa abordagem pra todas as demais automações:
- Cartão Comemorativo
- Cartão de Boas-vindas
- Cartão de Visita (Físico/Digital)
- Divulgação NPS
- Convite FP
- Certificado de Eventos

Pra solicitações **não-automação** (tipos genéricos que viram task no ClickUp), o detail view atual permanece — só muda a página das automações.

---

## Comportamento esperado

### Roteamento

URL atual provavelmente algo tipo `/solicitacoes/:id` que renderiza um detail view genérico.

Manter a URL. Dentro do handler/render da página, **detectar se o tipo da solicitação é uma automação** (tem template e produz artefato) e renderizar conforme:

- **Automação** → layout enxuto: preview + download (este prompt)
- **Não-automação** → layout atual (detalhes da request, status, etc)

Lista de tipos que devem usar o novo layout:

```js
const AUTOMATION_TYPES = [
  'assinatura-email',
  'cartao-boas-vindas',
  'cartao-comemorativo',
  'cartao-visita-fisico',
  'cartao-visita-digital',
  'divulgacao-nps',
  'convite-fp',
  'certificado-eventos',
];
```

Idealmente essa lista vem do FORM_SCHEMAS (qualquer form com `template_variant_field` ou que tenha template_renderer associado). Não vale duplicar hardcoded.

---

## Referência visual: assinatura de e-mail

Usar a página atual de assinatura de e-mail como template. Principais princípios visuais:

- Preview grande, centralizado no canvas
- Botão de download proeminente (dourado, mesmo padrão da Thank You)
- Sem campo "resumo" ou listagem dos dados preenchidos
- Sem timeline de status, sem comentários, sem metadados extras visíveis
- Header simples com nome do material e voltar
- Espaço respirando

### JS de controle

```js
window.SolicitacaoAutomacao = {
  solicitacaoId: null,
  pollInterval: null,
  formLabels: {
    'cartao-comemorativo':   'Cartão Comemorativo',
    'cartao-boas-vindas':    'Cartão de Boas-vindas',
    'assinatura-email':      'Assinatura de E-mail',
    'cartao-visita-fisico':  'Cartão de Visita Físico',
    'cartao-visita-digital': 'Cartão de Visita Digital',
    'divulgacao-nps':        'Material de Divulgação NPS',
    'convite-fp':            'Convite FP',
    'certificado-eventos':   'Certificado de Evento',
  },

  async init() {
    // Pegar ID da URL: /solicitacoes/:id ou ?id=
    const pathMatch = window.location.pathname.match(/\/solicitacoes\/(\d+)/);
    const params = new URLSearchParams(window.location.search);
    this.solicitacaoId = pathMatch?.[1] || params.get('id');
    if (!this.solicitacaoId) {
      this.renderError('Solicitação não encontrada.');
      return;
    }
    await this.refresh();
  },

  async refresh() {
    try {
      const res = await fetch(`/api/solicitacoes/${this.solicitacaoId}`, { credentials: 'include' });
      if (!res.ok) throw new Error('Falha ao buscar solicitação');
      const sol = await res.json();

      // Atualizar título
      const label = this.formLabels[sol.tipo] || 'Material';
      document.getElementById('solicitacaoTitle').textContent = label;

      if (sol.arquivo_gerado_url) {
        this.renderReady(sol);
        return;
      }
      if (sol.erro_geracao) {
        this.renderError(sol.erro_geracao, sol);
        return;
      }
      // Gerando — iniciar polling
      this.renderGenerating(sol);
      if (!this.pollInterval) {
        this.pollInterval = setInterval(() => this.refresh(), 2000);
      }
    } catch (err) {
      this.renderError(err.message);
    }
  },

  renderReady(sol) {
    if (this.pollInterval) { clearInterval(this.pollInterval); this.pollInterval = null; }
    document.getElementById('solicitacaoMain').innerHTML = `
      <div class="material-preview-wrapper">
        <img src="${this.escape(sol.arquivo_gerado_url)}" 
             alt="Preview do material"
             class="material-preview-img" />
      </div>
      <div class="material-download-area">
        <a href="${this.escape(sol.arquivo_gerado_url)}" 
           download 
           class="material-download-btn">
          ⬇ Baixar material
        </a>
      </div>
    `;
  },

  renderGenerating(sol) {
    document.getElementById('solicitacaoMain').innerHTML = `
      <div class="material-status">
        <div class="spinner"></div>
        <h2>Gerando seu material...</h2>
        <p class="material-status-hint">Isto pode levar alguns segundos. A página atualiza sozinha quando estiver pronto.</p>
      </div>
    `;
  },

  renderError(message, sol) {
    if (this.pollInterval) { clearInterval(this.pollInterval); this.pollInterval = null; }
    const canRetry = sol?.id;  // mostrar botão de regerar se temos a solicitação carregada
    document.getElementById('solicitacaoMain').innerHTML = `
      <div class="material-status warning">
        <div class="material-status-icon">⚠️</div>
        <h2>Não foi possível gerar o material</h2>
        <p>Nossa equipe foi notificada. Você pode tentar regerar:</p>
        ${canRetry ? `<button onclick="SolicitacaoAutomacao.tryAgain()" class="material-retry-btn">↻ Tentar regerar</button>` : ''}
        <details class="material-error-details">
          <summary>Detalhes técnicos</summary>
          <pre>${this.escape(message)}</pre>
        </details>
      </div>
    `;
  },

  async tryAgain() {
    try {
      const res = await fetch(`/api/admin/solicitacoes/${this.solicitacaoId}/regerar`, {
        method: 'POST', credentials: 'include'
      });
      if (!res.ok) throw new Error('Falha ao acionar regeração');
      await this.refresh();
    } catch (err) {
      alert('Não foi possível regerar agora.');
    }
  },

  escape(s) {
    return String(s ?? '').replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;').replace(/"/g,'&quot;');
  },
};
```

## Roteamento e roteador

No backend, o handler de rota da página de solicitação deve:

1. Receber o ID da URL (`/solicitacoes/:id`)
2. Buscar a solicitação no DB pra saber o tipo
3. Decidir qual HTML servir:
   - Se tipo está em `AUTOMATION_TYPES` (ou tem template associado): servir `solicitacao-automacao.html`
   - Senão: servir `solicitacao-detalhes.html` (página atual com resumo completo)

```ts
app.get('/solicitacoes/:id', async (req, res) => {
  const sol = await db.query.solicitacoes.findFirst({ where: eq(solicitacoes.id, req.params.id) });
  if (!sol) return res.status(404).sendFile('404.html');

  const isAutomation = AUTOMATION_TYPES.includes(sol.tipo);
  const filename = isAutomation ? 'solicitacao-automacao.html' : 'solicitacao-detalhes.html';
  res.sendFile(path.join(STATIC_DIR, filename));
});
```

Alternativa: servir sempre o mesmo HTML mas com lógica de roteamento client-side que detecta tipo e renderiza conforme. A primeira é mais simples.

## Validação

1. **Automação pronta**: ir na lista → clicar numa solicitação de Cartão Comemorativo concluída → ver página clean com preview e botão dourado de download. NÃO ver lista dos campos preenchidos no form.
2. **Automação ainda gerando**: criar solicitação de cartão com delay no render (simular 5s) → entrar na página → ver spinner → após 5s atualiza pra estado pronto sem refresh.
3. **Automação com erro**: forçar erro de render → entrar na página → ver mensagem clara, botão "Tentar regerar" (se admin) e detalhes técnicos no expansível.
4. **Não-automação**: ir numa solicitação de tipo genérico (que vai pro ClickUp) → ainda ver a página atual com resumo/detalhes/status. Esse fluxo NÃO foi alterado.
5. **Email signature**: confirmar que a página de assinatura de e-mail (referência visual) continua funcionando igual — esse prompt não deve quebrá-la.
6. **Header e shell**: a página usa Shell.render normalmente — admin vê sidebar à esquerda, usuário não.

---

## Considerações de UX

- **Tamanho do preview:** o material renderizado pode ter dimensões variadas (1080x1080 pro cartão, 600x200 pra assinatura). O CSS de `material-preview-img` usa `max-width: 100%; max-height: 600px` pra acomodar ambos sem distorcer.
- **Fundo cinza no preview:** ajuda a visualizar materiais com fundo branco/claro, que se confundiriam com o branco da página.
- **"Voltar"** usa `history.back()` por simplicidade — funciona pra quem chegou clicando na lista. Se quiser garantir destino fixo, trocar pra `window.location.href = '/solicitacoes'`.
