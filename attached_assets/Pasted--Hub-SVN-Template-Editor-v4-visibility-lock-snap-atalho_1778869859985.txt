## Etapa 1 — Visibility + Lock por layer

### 1.1 Schema

Adicionar duas flags opcionais ao `LayerBase`:

```ts
export type LayerBase = {
  id: string;
  name: string;
  x: number;
  y: number;
  visible?: boolean;    // default: true. Se false, layer é pulada no render.
  locked?: boolean;     // default: false. Se true, drag/resize desabilitados no editor.
};
```

### 1.2 Renderer

Em `renderFromTemplate`, pular layers com `visible === false`:

```ts
for (const layer of template.layers) {
  if (layer.visible === false) continue;
  // ... resto
}
```

### 1.3 UI — Lista de layers

Cada item da lista mostra dois ícones clicáveis à esquerda do nome:

```
👁  🔓  Logo da marca       image
🚫  🔓  Selos               image     ← visível=false (greyed out)
👁  🔒  Linha divisória     image     ← locked=true
```

- Click no olho → toggle `visible`
- Click no cadeado → toggle `locked`
- Layers ocultas: opacity 0.4 no item, label tachado
- Layers locked: ícone de cadeado preenchido

### 1.4 UI — Canvas overlay

- Layers ocultas: box NÃO renderizada no overlay (acompanha o render)
- Layers locked: box renderizada normalmente, mas com cursor `not-allowed` e sem interact.js drag/resize ativos
- Indicação visual: box locked tem borda pontilhada mais espessa + ícone de cadeado pequeno no canto da label

### 1.5 UI — Propriedades

Quando uma layer locked está selecionada:
- Inputs de X, Y, W, H desabilitados (greyed out + cursor not-allowed)
- Outros campos (fonte, cor, conteúdo) continuam editáveis
- Banner no topo do painel: "🔒 Layer bloqueada — desbloqueie para mover ou redimensionar"

### 1.6 Integração com interact.js

Em `setupInteract`:

```js
function setupInteract(el) {
  const layer = getLayer(el.dataset.id);
  if (!layer) return;
  if (layer.locked) {
    // Não anexar draggable/resizable
    el.style.cursor = 'not-allowed';
    return;
  }
  interact(el).draggable({...}).resizable({...});
}
```

Quando o usuário toggle `locked`, chamar `renderCanvas()` pra re-setup do interact.

## Etapa 2 — Snap & alignment guides

### 2.1 Comportamento esperado

Ao arrastar uma layer, mostrar **linhas guia** quando a posição se alinhar com:
- Centro vertical do canvas
- Centro horizontal do canvas
- Bordas do canvas (top, bottom, left, right)
- Edges (top/bottom/left/right) de outras layers visíveis
- Centros (vertical e horizontal) de outras layers visíveis

Quando dentro do threshold (5px de tolerância), **snap** pra essa posição automaticamente, e exibe linha guia magenta/verde na cor de destaque.

### 2.2 Implementação com interact.js

A `interact.js` tem o modifier `snap` nativo. Usar:

```js
function getSnapTargets(currentLayerId) {
  const SCALE = getScale();
  const W = currentTemplate.canvas.width * SCALE;
  const H = currentTemplate.canvas.height * SCALE;
  const targets = [
    // Centros do canvas
    { x: W / 2 },
    { y: H / 2 },
    // Bordas
    { x: 0 }, { x: W },
    { y: 0 }, { y: H },
  ];
  // Edges e centros de outras layers
  for (const layer of currentTemplate.layers) {
    if (layer.id === currentLayerId) continue;
    if (layer.visible === false) continue;
    const lx = layer.x * SCALE;
    const ly = layer.y * SCALE;
    const lw = (layer.w || 0) * SCALE;
    const lh = (layer.h || 0) * SCALE;
    targets.push({ x: lx }, { x: lx + lw }, { x: lx + lw / 2 });
    targets.push({ y: ly }, { y: ly + lh }, { y: ly + lh / 2 });
  }
  return targets;
}

interact(el).draggable({
  modifiers: [
    interact.modifiers.snap({
      targets: getSnapTargets(layer.id),
      range: 6,  // px threshold
      relativePoints: [
        { x: 0, y: 0 },     // top-left
        { x: 0.5, y: 0.5 }, // center
        { x: 1, y: 1 },     // bottom-right
      ],
    }),
  ],
  // ... outros listeners
});
```

### 2.3 Linhas guia visuais

Criar um overlay de SVG fixo dentro de `canvasWrap` que renderiza as linhas guia ativas:

```html
<svg class="snap-guides" id="snapGuides"></svg>
```

```css
.snap-guides {
  position: absolute; inset: 0; pointer-events: none; z-index: 999;
}
.snap-guides line {
  stroke: #f0a; stroke-width: 1; stroke-dasharray: 3,3;
}
```

Durante o drag, listener `move` do interact verifica se houve snap (compara posição atual com targets dentro do range) e adiciona linhas no SVG. No `end`, limpa o SVG.

### 2.4 Toggle de snap

Botão na toolbar do canvas:

```
[ Snap: ON ]  ← clicável, alterna ON/OFF
```

Default: ON. Quando OFF, snap não é aplicado e guides não aparecem. Persistir preferência em `localStorage`.

## Etapa 3 — Atalhos de teclado

### 3.1 Atalhos a implementar

```js
document.addEventListener('keydown', (e) => {
  // Ignora se foco está em input/textarea
  const tag = e.target.tagName;
  if (tag === 'INPUT' || tag === 'TEXTAREA' || tag === 'SELECT') return;

  const meta = e.ctrlKey || e.metaKey;

  // Delete = deletar layer selecionada
  if ((e.key === 'Delete' || e.key === 'Backspace') && selectedLayerId) {
    e.preventDefault();
    deleteLayer(selectedLayerId);
  }

  // Cmd/Ctrl + D = duplicar layer
  if (meta && e.key.toLowerCase() === 'd' && selectedLayerId) {
    e.preventDefault();
    duplicateLayer(selectedLayerId);
  }

  // Cmd/Ctrl + S = salvar
  if (meta && e.key.toLowerCase() === 's') {
    e.preventDefault();
    saveTemplate();
  }

  // Esc = desselecionar
  if (e.key === 'Escape') {
    e.preventDefault();
    selectLayer(null);
  }

  // Setas = nudge
  if (['ArrowUp', 'ArrowDown', 'ArrowLeft', 'ArrowRight'].includes(e.key) && selectedLayerId) {
    e.preventDefault();
    const step = e.shiftKey ? 10 : 1;
    const layer = getLayer(selectedLayerId);
    if (!layer || layer.locked) return;
    if (e.key === 'ArrowUp')    layer.y -= step;
    if (e.key === 'ArrowDown')  layer.y += step;
    if (e.key === 'ArrowLeft')  layer.x -= step;
    if (e.key === 'ArrowRight') layer.x += step;
    markDirty();
    renderCanvas();
    showProps(selectedLayerId);
    pushHistory(); // se v3 já implementou undo
  }
});
```

### 3.2 Duplicar layer

```js
function duplicateLayer(id) {
  const layer = getLayer(id);
  if (!layer) return;
  const copy = JSON.parse(JSON.stringify(layer));
  copy.id = `${layer.id}_copy_${Date.now()}`;
  copy.name = `${layer.name} (cópia)`;
  copy.x = (copy.x || 0) + 20;
  copy.y = (copy.y || 0) + 20;
  currentTemplate.layers.push(copy);
  markDirty();
  renderCanvas();
  renderLayersList();
  selectLayer(copy.id);
}
```

## Etapa 4 — Conditional rendering

### 4.1 Schema

```ts
export type LayerBase = {
  // ...campos existentes
  show_when?: ShowCondition;
};

export type ShowCondition = {
  field: string;        // ex: 'tem_cfp'
  operator: 'equals' | 'not_equals' | 'truthy' | 'falsy';
  value?: any;          // pro equals e not_equals
};
```

### 4.2 Renderer

Em `renderFromTemplate`, antes de cada layer:

```ts
function shouldRenderLayer(layer: Layer, data: Record<string, string>): boolean {
  if (layer.visible === false) return false;
  if (!layer.show_when) return true;
  const { field, operator, value } = layer.show_when;
  const fieldValue = data[field];
  switch (operator) {
    case 'equals':     return fieldValue == value;
    case 'not_equals': return fieldValue != value;
    case 'truthy':     return !!fieldValue && fieldValue !== 'false' && fieldValue !== 'nao';
    case 'falsy':      return !fieldValue || fieldValue === 'false' || fieldValue === 'nao';
  }
}

for (const layer of template.layers) {
  if (!shouldRenderLayer(layer, dataStr)) continue;
  // ... renderiza
}
```

### 4.3 UI nas propriedades da layer

Seção "Renderização condicional" no painel direito:

```
┌─ Renderização condicional ─────────────┐
│ ☐ Renderizar apenas se condição match  │
│                                         │
│ (quando marcado:)                       │
│ Campo:     [tem_cfp        ▾]          │
│ Operador:  [equals         ▾]          │
│ Valor:     [sim__________________]     │
│                                         │
│ Atualmente: vai renderizar (✓)         │
└─────────────────────────────────────────┘
```

- Checkbox liga/desliga a condição
- Dropdown de "Campo" usa os placeholders do tipo (PLACEHOLDERS_BY_TIPO)
- "Atualmente" mostra se a layer está renderizando com os `testData` atuais

### 4.4 Casos de uso pra documentar internamente

- **Selo CFP** na assinatura → `show_when: { field: 'tem_cfp', operator: 'equals', value: 'sim' }`
- **Cargo** na assinatura → `show_when: { field: 'cargo', operator: 'truthy' }` (renderiza só se tiver cargo preenchido)
- **Layer extra "Cliente Private"** → `show_when: { field: 'is_private_key', operator: 'equals', value: 'private' }`

Isso permite que esses casos sejam configurados via UI ao invés de hardcoded em handlers.

## Etapa 5 — Fallback de erro em produção

### 5.1 Problema

Hoje, se um template tem bug que faz Sharp crashar, o handler do form retorna 500 e o usuário não consegue submeter. Solicitação não é criada, dados são perdidos.

Objetivo: garantir que **submissão de form NUNCA falha por causa de problema de geração de arte**.

### 5.2 Padrão de fallback

Em cada handler que chama `renderFromTemplate`:

```ts
async function handleSolicitacao(req, res) {
  // ... validações e criação da solicitação no DB ...
  const solicitacao = await db.insert(solicitacoes).values({...}).returning();

  // Geração da arte — best effort, não bloqueia
  try {
    const template = await db.query.artTemplates.findFirst({
      where: and(
        eq(artTemplates.tipo, tipo),
        eq(artTemplates.is_active, true)
      ),
    });
    if (!template) throw new Error('Template ativo não encontrado');

    const pngBuffer = await renderFromTemplate(template.config, dados);
    const url = await uploadToR2(`${tipo}/${solicitacao.id}.png`, pngBuffer, 'image/png');
    await db.update(solicitacoes)
      .set({ artefato_url: url })
      .where(eq(solicitacoes.id, solicitacao.id));
    webhookPayload.artefatoUrl = url;
  } catch (err) {
    // Logar com contexto completo
    req.log.error({
      err,
      template_tipo: tipo,
      template_id: template?.id,
      solicitacao_id: solicitacao.id,
      dados,
    }, 'Falha na geração de arte');

    // Marcar a solicitação com status especial
    await db.update(solicitacoes)
      .set({
        artefato_url: null,
        artefato_erro: err.message,
        artefato_status: 'falhou',
      })
      .where(eq(solicitacoes.id, solicitacao.id));

    // Notificar admin (ver 5.4)
    await notifyAdminTemplateError({
      tipo,
      template_id: template?.id,
      solicitacao_id: solicitacao.id,
      error_message: err.message,
    });

    // Webhook payload sem URL — N8N pode reagir ou só processar sem arte
    webhookPayload.artefatoUrl = null;
    webhookPayload.artefatoFalhou = true;
  }

  // Webhook acontece independente do sucesso da arte
  await sendWebhook(webhookPayload);

  res.json({ id: solicitacao.id, success: true });
}
```

### 5.3 Schema — coluna pra rastrear falhas

```sql
ALTER TABLE solicitacoes ADD COLUMN IF NOT EXISTS artefato_erro TEXT;
ALTER TABLE solicitacoes ADD COLUMN IF NOT EXISTS artefato_status VARCHAR(20) DEFAULT 'pendente';
-- valores: 'pendente', 'sucesso', 'falhou'
```

E quando dá sucesso, atualizar `artefato_status = 'sucesso'`.

### 5.5 UI — Aba "Solicitações com erro" no dashboard admin

Adicionar filtro na lista de solicitações: "Mostrar apenas com erro de arte" (`artefato_status = 'falhou'`).

Cada linha mostra:
- ID da solicitação

### 5.6 Endpoint de retry

```ts
// POST /api/admin/solicitacoes/:id/regenerar-artefato
async function regenerarArtefato(req, res) {
  const { id } = req.params;
  const solicitacao = await db.query.solicitacoes.findFirst({ where: eq(solicitacoes.id, id) });
  if (!solicitacao) return res.status(404).json({ error: 'Solicitação não encontrada' });

  // Re-monta dados e tenta de novo
  const dados = JSON.parse(solicitacao.dados_json);
  try {
    const template = await db.query.artTemplates.findFirst({
      where: and(eq(artTemplates.tipo, solicitacao.tipo), eq(artTemplates.is_active, true)),
    });
    const pngBuffer = await renderFromTemplate(template.config, dados);
    const url = await uploadToR2(`${solicitacao.tipo}/${solicitacao.id}.png`, pngBuffer, 'image/png');
    await db.update(solicitacoes)
      .set({ artefato_url: url, artefato_erro: null, artefato_status: 'sucesso' })
      .where(eq(solicitacoes.id, solicitacao.id));
    res.json({ ok: true, url });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
}
```

### Smoke tests

1. **Visibility**: editor → toggle olho de uma layer → desaparece do canvas → salvar → renderizar → PNG sai sem essa layer.
2. **Lock**: toggle cadeado → tentar arrastar → não move. Inputs X/Y desabilitados no painel.
3. **Snap**: arrastar uma layer perto do centro do canvas → linha guia magenta aparece + snap pra posição.
4. **Atalhos**: selecionar layer, pressionar Delete → some. Cmd/Ctrl+D → duplica. Setas → move 1px. Shift+seta → 10px.
5. **Conditional rendering**: configurar layer "Selo CFP" com `show_when: tem_cfp equals sim`. Mudar testData pra `tem_cfp=nao` → preview some o selo. Voltar pra sim → reaparece.
6. **Fallback**: editar template com bug de propósito (ex.: setar layer image w=99999) → salvar → submeter form pelo fluxo normal → form completa com sucesso, solicitação criada com `artefato_status='falhou'`, log no servidor mostra erro detalhado.
7. **Retry**: ir na aba admin de solicitações, filtrar pelas com falha, corrigir o template (mudar w pra valor válido), clicar "Tentar novamente" → arte gera com sucesso.

### Considerações de migração

- Templates existentes não têm `visible`/`locked`/`show_when` → defaults `undefined` (que significa `true`/`false`/sem condição). Backward compatible.
- Coluna `artefato_status` recém-criada → default `pendente` pra rows novas. Pra rows antigas pode ficar NULL, ou rodar UPDATE pra setar `sucesso` onde `artefato_url IS NOT NULL`.
