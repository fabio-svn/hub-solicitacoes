## Item 6 — Filtro "Em uso" funcional em admin-assets.html

No arquivo admin-assets.html, o select de filtro tem 3 opções:
- "" (todos)
- "orphan" (não utilizados)
- "used" (em uso)

Mas o JS só mapeia 'orphan' pro endpoint:
  if (usage === 'orphan') url += 'orphan=1&';

Filtrar por "Em uso" hoje retorna TODOS os assets em vez de só os 
em uso.

Ação:
Verificar primeiro se o backend src/routes/assets.ts aceita um 
parâmetro de filtro inverso (ex: ?used=1).

- Se SIM: adicionar no JS:
    if (usage === 'used') url += 'used=1&';

- Se NÃO: filtrar no frontend após receber a resposta. Adicionar 
  depois do filtro existente:
    if (usage === 'used') {
      allAssets = allAssets.filter(a => a.usage_count > 0);
    }
