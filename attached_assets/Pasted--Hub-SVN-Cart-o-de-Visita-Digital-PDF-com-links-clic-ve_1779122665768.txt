# Hub SVN — Cartão de Visita Digital: PDF com links clicáveis + borda circular

## Contexto e escopo

Implementar o tipo de automação **Cartão de Visita Digital**, que requer:

1. **Saída em PDF** (não PNG)
2. **Links clicáveis** em layers específicas:
   - Telefone → URL do WhatsApp
   - E-mail → `mailto:`
   - Logo → site institucional da marca
3. **Imagem circular com borda** pra foto do solicitante
4. **Logo variando por contrato social** (SVN Investimentos, Connect, Capital)

Esse prompt cobre 4 etapas: core de PDF, links, borda/shape, e o form específico. Cada etapa adiciona feature genérica que servirá pra outros templates futuros (certificado, etc.).

## Etapa 1 — Renderer PDF (genérico)

### 1.1 Dependência

```bash
cd artifacts/api-server && pnpm add pdf-lib
```

### 1.2 Novo serviço: `src/services/pdf-renderer.ts`

```ts
import { PDFDocument, rgb, StandardFonts } from 'pdf-lib';

export async function renderTemplateToPdf(template, data) {
  const pdf = await PDFDocument.create();
  const page = pdf.addPage([template.canvas.width, template.canvas.height]);

  // 1. Background
  await drawBackground(pdf, page, template.bg, data);

  // 2. Layers (na ordem do array, primeiro = atrás)
  for (const layer of template.layers) {
    if (!layer.visible) continue;
    
    if (layer.type === 'image') {
      await drawImageLayer(pdf, page, layer, data);
    } else {  // text-line, text-block
      await drawTextLayer(pdf, page, layer, data);
    }
  }

  return await pdf.save();  // retorna Uint8Array
}
```

Funções auxiliares (`drawBackground`, `drawImageLayer`, `drawTextLayer`) seguem mesma lógica do renderer PNG existente, mas usando API do pdf-lib.

### 1.3 Campo `output_format` no template

Schema da config do template ganha campo opcional:

```ts
type ArtTemplate = {
  tipo: string;
  output_format?: 'png' | 'pdf';  // default: 'png' (compat)
  canvas: { width, height };
  bg: {...};
  layers: [...];
};
```

### 1.4 Editor: seletor de formato

Na aba "Documento" do editor, adicionar:

```
Formato de saída:
( ) PNG (imagem)
(●) PDF (com suporte a links clicáveis)
```

Salva em `template.config.output_format`.

### 1.5 Handler de solicitação

Em `/api/solicitacoes`, após carregar template:

```ts
let buffer, mimeType, extension;
if (template.config.output_format === 'pdf') {
  buffer = await renderTemplateToPdf(template.config, data);
  mimeType = 'application/pdf';
  extension = 'pdf';
} else {
  buffer = await renderFromTemplate(template.config, data);
  mimeType = 'image/png';
  extension = 'png';
}

const storageKey = `geradas/${tipo}/${id}-${Date.now()}.${extension}`;
const url = await uploadToR2(storageKey, buffer, mimeType);
```

## Etapa 2 — Links clicáveis em layers

### 2.1 Schema novo: `link` em layers

Qualquer layer (texto ou imagem) ganha campo opcional `link`:

```ts
type LayerLink = 
  | { type: 'static',      url: string }
  | { type: 'template',    template: string }    // ex: 'https://wa.me/{{telefone_digits}}'
  | { type: 'placeholder', placeholder: string }; // valor direto do campo

type Layer = {
  // ... campos existentes
  link?: LayerLink;
};
```

### 2.2 Editor: painel de configuração de link

No painel de propriedades de uma layer (texto ou imagem), seção "Link clicável":

```
┌─ Link clicável ──────────────────────────────┐
│ [ ☐ Tornar este elemento clicável ]          │
│                                              │
│ Tipo de link: [template ▾]                   │
│   ├─ static       (URL fixa)                 │
│   ├─ template     (URL com placeholders)     │
│   └─ placeholder  (valor de campo é o URL)   │
│                                              │
│ Template:                                    │
│ [https://wa.me/{{telefone_digits}}      ]    │
│                                              │
│ Hint: placeholders disponíveis pra este form │
│ • {{nome}}                                   │
│ • {{telefone}}                               │
│ • {{telefone_digits}}                        │
│ • {{email}}                                  │
│ • ...                                        │
└──────────────────────────────────────────────┘
```

A lista de placeholders vem do FORM_SCHEMAS do tipo do template (campos + computed).

### 2.3 PDF renderer aplica link como annotation

Em `pdf-renderer.ts`, após desenhar a layer, se tiver `link`:

```ts
import { PDFName, PDFArray, PDFString } from 'pdf-lib';

function applyLinkAnnotation(page, layer, resolvedUrl) {
  const annotation = page.doc.context.obj({
    Type: 'Annot',
    Subtype: 'Link',
    Rect: [
      layer.x,
      page.getHeight() - layer.y - layer.height,  // PDF coords invertidas
      layer.x + layer.width,
      page.getHeight() - layer.y,
    ],
    Border: [0, 0, 0],  // sem borda visual
    A: {
      Type: 'Action',
      S: 'URI',
      URI: PDFString.of(resolvedUrl),
    },
  });
  
  let annotations = page.node.get(PDFName.of('Annots'));
  if (!annotations) {
    annotations = page.doc.context.obj([]);
    page.node.set(PDFName.of('Annots'), annotations);
  }
  annotations.push(annotation);
}
```

### 2.4 Resolução de link na renderização

Antes de aplicar a annotation, resolver placeholders no URL:

```ts
function resolveLink(link, data) {
  if (link.type === 'static') return link.url;
  if (link.type === 'placeholder') return data[link.placeholder] || '';
  if (link.type === 'template') {
    return link.template.replace(/\{\{(\w+)\}\}/g, (_, key) => data[key] || '');
  }
}
```

## Etapa 3 — Computed fields no FORM_SCHEMAS

Pra que `telefone_digits` (e outros campos derivados) funcionem, o schema precisa declarar e o backend precisa calcular antes da renderização.

### 3.1 Schema com computed

```ts
type ComputedField = {
  name: string;
  label: string;
  derived_from: string;
  transform: 'digits_only' | 'whatsapp_link' | 'mailto_link' | 'website_by_value' | 'identity';
  // opcional: tabela de lookup
  lookup?: Record<string, string>;
};

// Exemplo em assinatura-email ou cartao-visita-digital:
'cartao-visita-digital': {
  tipo: 'cartao-visita-digital',
  label: 'Cartão de Visita Digital',
  template_variant_field: 'contrato_social',
  fields: [
    { name: 'nome',     label: 'Nome',     type: 'text' },
    { name: 'telefone', label: 'Telefone', type: 'tel' },
    { name: 'email',    label: 'E-mail',   type: 'email' },
    { name: 'foto_url', label: 'Foto (URL após upload)', type: 'text', hidden: true },
    {
      name: 'contrato_social', label: 'Contrato Social', type: 'select',
      options: [
        { value: 'svn-investimentos', label: 'SVN Investimentos' },
        { value: 'svn-connect',       label: 'SVN Connect' },
        { value: 'svn-capital',       label: 'SVN Capital' },
      ],
    },
  ],
  computed: [
    { name: 'telefone_digits', label: 'Telefone (só dígitos com 55)',
      derived_from: 'telefone', transform: 'digits_only' },
    { name: 'site_url', label: 'URL do site da marca',
      derived_from: 'contrato_social', transform: 'website_by_value',
      lookup: {
        'svn-investimentos': 'https://svninvestimentos.com.br',
        'svn-connect':       'https://svnconnect.com.br',
        'svn-capital':       'https://svncapital.com.br',
      } 
    },
  ],
}
```

### 3.2 Backend resolve computed antes de renderizar

Em `processarSolicitacao` (ou nome equivalente), antes de chamar o renderer:

```ts
function resolveComputed(payload, schema) {
  const result = { ...payload };
  for (const c of schema.computed || []) {
    const source = payload[c.derived_from];
    if (!source) continue;
    
    switch (c.transform) {
      case 'digits_only':
        result[c.name] = source.replace(/\D/g, '');
        // Adiciona prefixo 55 se Brasil e não começa com 55
        if (result[c.name].length === 11 && !result[c.name].startsWith('55')) {
          result[c.name] = '55' + result[c.name];
        }
        break;
      case 'website_by_value':
        result[c.name] = c.lookup?.[source] || '';
        break;
      // ...outros transforms
    }
  }
  return result;
}

const data = resolveComputed(payload, FORM_SCHEMAS[tipo]);
await renderTemplateToPdf(template.config, data);
```

## Etapa 4 — Shape (círculo) + Border em layers de imagem

### 4.1 Schema novo

```ts
type ImageLayer = {
  type: 'image',
  source: ...,
  shape?: 'rectangle' | 'circle';   // default: 'rectangle'
  border?: {
    width: number;    // em pixels
    color: string;    // hex, ex: '#AC3631'
  };
  // ... campos existentes
};
```

### 4.2 Editor: painel de propriedades de layer image

Adicionar:

```
Forma:
(●) Retângulo
( ) Círculo

Borda:
  Largura: [ 4 ] px
  Cor:     [ #AC3631 ] 🎨
  (deixar 0 pra sem borda)
```

Quando círculo é escolhido, esconder ou desabilitar campos de "border-radius" se existirem (círculo é redondo perfeito, não retângulo arredondado).

Pra círculo, garantir que `width === height` no canvas (avisar usuário se diferente, ou forçar quadrado).

### 4.3 Renderer: imagem circular + borda

**PNG renderer (Sharp):**

```ts
async function drawImageLayerPng(canvas, layer, data) {
  let imgBuffer = await loadImage(resolveSource(layer.source, data));
  imgBuffer = await sharp(imgBuffer).resize(layer.width, layer.height).toBuffer();
  
  if (layer.shape === 'circle') {
    // Criar máscara circular usando SVG
    const mask = Buffer.from(`
      <svg width="${layer.width}" height="${layer.height}">
        <circle cx="${layer.width / 2}" cy="${layer.height / 2}" 
                r="${layer.width / 2}" fill="white"/>
      </svg>
    `);
    imgBuffer = await sharp(imgBuffer)
      .composite([{ input: mask, blend: 'dest-in' }])
      .png()
      .toBuffer();
  }
  
  // Compor no canvas principal
  await canvas.composite([{ input: imgBuffer, left: layer.x, top: layer.y }]);
  
  // Borda (se tiver) desenhada por cima
  if (layer.border && layer.border.width > 0) {
    const borderSvg = buildBorderSvg(layer);
    await canvas.composite([{ input: Buffer.from(borderSvg), left: layer.x, top: layer.y }]);
  }
}
```

**PDF renderer (pdf-lib):**

```ts
async function drawImageLayerPdf(page, layer, data) {
  const imgBuffer = await loadImage(resolveSource(layer.source, data));
  const img = await page.doc.embedPng(imgBuffer);
  const y = page.getHeight() - layer.y - layer.height;
  
  if (layer.shape === 'circle') {
    // Salva graphic state, clipa pra círculo, desenha imagem, restaura state
    page.pushOperators(
      pushGraphicsState(),
      // clip path circular
      ...buildCirclePath(layer.x + layer.width/2, y + layer.height/2, layer.width/2),
      clip(),
      endPath(),
    );
    page.drawImage(img, { x: layer.x, y, width: layer.width, height: layer.height });
    page.pushOperators(popGraphicsState());
  } else {
    page.drawImage(img, { x: layer.x, y, width: layer.width, height: layer.height });
  }
  
  if (layer.border && layer.border.width > 0) {
    if (layer.shape === 'circle') {
      page.drawCircle({
        x: layer.x + layer.width/2,
        y: y + layer.height/2,
        size: layer.width/2,
        borderColor: rgb.fromHex(layer.border.color),
        borderWidth: layer.border.width,
      });
    } else {
      page.drawRectangle({
        x: layer.x, y, width: layer.width, height: layer.height,
        borderColor: rgb.fromHex(layer.border.color),
        borderWidth: layer.border.width,
      });
    }
  }
}
```

(Os helpers `pushGraphicsState`, `clip`, etc. vêm de `pdf-lib`. Verificar API exata na hora de implementar.)

## Etapa 5 — Form HTML: upload de foto

O form de Cartão de Visita Digital precisa de campo de upload de foto:

```html
<div class="field">
  <label>Sua foto *</label>
  <input type="file" id="fotoInput" accept="image/*" onchange="uploadFoto(this.files[0])" />
  <div id="fotoPreview" style="display:none">
    <img id="fotoPreviewImg" />
    <button onclick="removeFoto()">Remover</button>
  </div>
</div>

<input type="hidden" id="fotoUrl" name="foto_url" />
```

```js
async function uploadFoto(file) {
  // Upload pro endpoint existente de assets
  const formData = new FormData();
  formData.append('file', file);
  const res = await fetch('/api/admin/assets/upload', { method: 'POST', body: formData });
  const { url } = await res.json();
  
  // Setar URL no campo hidden
  document.getElementById('fotoUrl').value = url;
  
  // Preview
  document.getElementById('fotoPreviewImg').src = url;
  document.getElementById('fotoPreview').style.display = 'block';
}
```

Nota: o upload pode usar o mesmo endpoint do asset library, mas marcar de alguma forma que é "foto de solicitante" (não asset reutilizável). Ou criar endpoint separado `/api/uploads/foto-solicitante` que não cria entry na tabela `art_assets` (já que essas fotos são uso único).

## Considerações

- **Tamanho do PDF gerado**: provavelmente 200-500KB com foto + logo
- **Performance**: pdf-lib é rápido, geração leva < 1s
- **Compatibilidade do renderer PNG**: layers com `shape='circle'` ou `border` também funcionam no renderer PNG existente. Layers sem essas props continuam exatamente como antes.
- **Templates antigos**: nada quebra. Os campos `shape`, `border`, `link` são opcionais. Templates sem eles seguem comportamento anterior.
