# Hub SVN — Ajustes UI do Template Editor

5 ajustes pontuais. Cada um afeta uma área específica.

## #1 — Um header só (remover o duplicado)

**Problema:** existe um header próprio do editor de templates + o header padrão do hub. Ficou duplicado.

**Correção:**

1. **Remover o header customizado** do editor (provavelmente um `<header class="editor-header">` ou similar no início do `admin-templates.html`).

2. **Adicionar o botão "Voltar" ao header padrão** que já existe (renderizado por `Auth.renderHeader()`). Duas formas de fazer isso:

**Opção A — Estender `Auth.renderHeader` pra aceitar config opcional:**

```js
Auth.renderHeader(headerEl, {
  backButton: {
    label: '← Voltar',
    onClick: () => handleVoltar()  // ver item #4
  }
});
```

`Auth.renderHeader` insere o botão à esquerda do logo SVN. Se não receber config, comporta-se como antes.

**Opção B (mais simples se A for muito invasivo) — Injetar após o render:**

No `admin-templates.html`, depois de `Auth.renderHeader(document.getElementById('mainHeader'))`:

```js
const backBtn = document.createElement('button');
backBtn.className = 'header-back-btn';
backBtn.innerHTML = '← Voltar';
backBtn.onclick = handleVoltar;
// Inserir como primeiro filho do header, antes do logo
document.getElementById('mainHeader').prepend(backBtn);
```

CSS pra estilizar consistente com o header:
```css
.header-back-btn {
  background: transparent;
  border: 1px solid rgba(0,0,0,0.1);
  padding: 6px 14px;
  border-radius: 6px;
  font-family: 'Nunito Sans', sans-serif;
  font-size: 0.85rem;
  cursor: pointer;
  color: #221b19;
}
.header-back-btn:hover { background: rgba(0,0,0,0.04); }
```

A Opção A é a correta a longo prazo (centraliza no Auth, fácil de usar em outras telas), mas Opção B resolve mais rápido se o Auth.renderHeader for complexo de mexer.

## #2 — Zoom afeta só o artboard, não a página inteira

**Problema atual:** ao dar zoom, o canvas inteiro cresce e empurra o resto da página, criando scroll global. Não tem sentido — o canvas deveria ser uma "viewport" fixa onde o artboard é escalado dentro.

**Correção em CSS + JS:**

### CSS

A estrutura deve ser:

```css
/* O contêiner do canvas: TAMANHO FIXO baseado no viewport */
.canvas-scroll {
  flex: 1;                      /* ocupa o espaço disponível */
  height: 100%;                 /* não cresce além do que o pai define */
  overflow: auto;               /* gera scroll interno se o artboard for maior */
  background: #2a2a2a;
  border-radius: 12px;
  position: relative;
  display: flex;
  align-items: flex-start;
  justify-content: center;
  padding: 24px;
}

/* O artboard: cresce conforme zoom, mas DENTRO do canvas-scroll */
.canvas-wrap {
  position: relative;
  display: inline-block;
  flex-shrink: 0;
  /* width definido via JS conforme canvasScale */
}
```

A chave é o `overflow: auto` no `canvas-scroll`. Quando o `canvas-wrap` for maior que o `canvas-scroll`, o scroll vai aparecer dentro da área escura — sem afetar o resto da página.

### JS

Garantir que `applyCanvasScale` está setando largura no `canvas-wrap` (não em outro elemento):

```js
function applyCanvasScale() {
  const wrap = document.getElementById('canvasWrap');
  if (!wrap || !currentTemplate) return;
  wrap.style.width = (currentTemplate.canvas.width * canvasScale) + 'px';
  // NÃO setar height — a img bg-img com `height: auto` já cuida
  renderCanvas();
  updateZoomLabel();
}
```

E garantir que o **layout pai do canvas-scroll** (provavelmente `.canvas-panel`) tem `display: flex; flex-direction: column; height: 100%` ou similar, pra que o `canvas-scroll` herde altura disponível corretamente.

### Resultado

- Zoom in: scroll aparece dentro da área escura, página fora não mexe
- Zoom out: artboard fica menor que o canvas-scroll, centraliza
- Fit-to-view: artboard cabe exatamente sem scroll

## #4 — Confirmação de "Voltar" ANTES de sair

**Problema:** ao clicar Voltar com mudanças não salvas, a página sai imediatamente e o popup só aparece depois. Cancelar deixa o usuário num estado quebrado.

**Causa provável:** o handler de Voltar está fazendo `window.history.back()` ou `window.location.href = ...` SEM verificar `isDirty` primeiro. O modal de confirmação está em outro lugar do código que dispara tarde demais.

**Correção:**

```js
function handleVoltar() {
  if (!isDirty) {
    navigateToList();
    return;
  }

  // Mostra modal de confirmação ANTES de navegar
  showUnsavedModal({
    onSaveAndLeave: async () => {
      const ok = await saveTemplate();
      if (ok) navigateToList();
    },
    onLeaveWithoutSaving: () => {
      isDirty = false;  // pra não disparar o beforeunload
      navigateToList();
    },
    onCancel: () => {
      closeUnsavedModal();
      // permanece na página, nada muda
    },
  });
}

function navigateToList() {
  window.location.href = '/admin/templates';
  // ou se for state-based: setView('list');
}

function showUnsavedModal({ onSaveAndLeave, onLeaveWithoutSaving, onCancel }) {
  const modal = document.createElement('div');
  modal.className = 'modal-overlay';
  modal.id = 'unsavedModal';
  modal.innerHTML = `
    <div class="modal-content modal-medium">
      <div class="modal-header">
        <h2>Mudanças não salvas</h2>
      </div>
      <div class="modal-body">
        <p>Se você sair agora, suas alterações serão perdidas.</p>
      </div>
      <div class="modal-footer">
        <button class="btn-tertiary" onclick="document.getElementById('unsavedModal').dispatchEvent(new CustomEvent('leave-without-saving'))">Sair sem salvar</button>
        <button class="btn-secondary" onclick="document.getElementById('unsavedModal').dispatchEvent(new CustomEvent('cancel'))">Cancelar</button>
        <button class="btn-primary" onclick="document.getElementById('unsavedModal').dispatchEvent(new CustomEvent('save-and-leave'))">Salvar e sair</button>
      </div>
    </div>
  `;
  document.body.appendChild(modal);
  modal.addEventListener('leave-without-saving', () => { closeUnsavedModal(); onLeaveWithoutSaving(); });
  modal.addEventListener('cancel', () => { closeUnsavedModal(); onCancel?.(); });
  modal.addEventListener('save-and-leave', () => { closeUnsavedModal(); onSaveAndLeave(); });
}

function closeUnsavedModal() {
  document.getElementById('unsavedModal')?.remove();
}
```

**Garantir que o botão Voltar do header (item #1) usa `handleVoltar` como handler.** E que NÃO há outro click handler concorrente em `<a href>` que navega antes da função rodar.

Se o Voltar for um link `<a>`, ele precisa virar `<button>` ou ter o `onclick` retornando `false` corretamente:

```html
<button onclick="handleVoltar()">← Voltar</button>
```

## #5 — Botão "Ajustar à tela": fixar e mostrar como texto

**Problema:**
- Não funciona (clica e não faz nada, ou faz a coisa errada)
- Está com ícone "⊡"; quer trocar pra texto

### 5.1 Trocar ícone por texto

No componente flutuante de zoom (canto inferior central do canvas), substituir:

```html
<button onclick="fitToView()" title="Ajustar à tela">⊡</button>
```

Por:

```html
<button onclick="fitToView()" class="zoom-fit-btn">Ajustar à tela</button>
```

Ajustar CSS pra acomodar texto:

```css
.zoom-floating button.zoom-fit-btn {
  padding: 4px 10px;
  font-size: 0.78rem;
  font-family: 'Nunito Sans', sans-serif;
  white-space: nowrap;
}
```

### 5.2 Fazer funcionar

A função `fitToView` deve calcular o scale que faz o artboard caber inteiro no `canvas-scroll` (descontando padding):

```js
function fitToView() {
  if (!currentTemplate) return;
  const scrollEl = document.getElementById('canvasScroll');
  if (!scrollEl) return;

  // Tamanho útil dentro do scroll (descontando padding interno)
  const availW = scrollEl.clientWidth - 48;   // 24px de cada lado
  const availH = scrollEl.clientHeight - 48;

  if (availW <= 0 || availH <= 0) return;

  const sW = availW / currentTemplate.canvas.width;
  const sH = availH / currentTemplate.canvas.height;

  // Pega o menor pra caber inteiro, e limita em 1 pra não ampliar
  canvasScale = Math.min(sW, sH, 1);
  applyCanvasScale();
}
```

Conferir que:
- `applyCanvasScale` está realmente atualizando `wrap.style.width`
- `updateZoomLabel` é chamado pra atualizar o "% atual" no display
- Não há outro JS sobrescrevendo `canvasScale` logo depois

Se mesmo após o fix o botão não responder, conferir no DevTools Console se a função existe globalmente (`window.fitToView`) — pode ser que o JS esteja dentro de um módulo isolado e o `onclick` no HTML não consegue alcançar.
