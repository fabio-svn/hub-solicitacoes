## Contexto

Hoje o art-generator pula direto de status="recebido" para "concluido" 
quando termina a geração. Se algo falha no meio (variant não encontrado, 
R2 fora, erro de fonte), o status NÃO vira "erro" — fica em "recebido" 
pra sempre. UI fica em "Processando..." sem feedback.

Esta tarefa adiciona estados explícitos: recebido → gerando → 
concluido/erro.

- artifacts/api-server/src/services/art-generator.ts
- artifacts/api-server/public/config.js
- (verificar) lib/db/src/schema/index.ts

## Tarefa 3 — Status flow no art-generator

Em artifacts/api-server/src/services/art-generator.ts, refatorar a 
função principal (geralmente generateArt ou similar) para:

  export async function generateArt(solicitacaoId: number) {
    // 1. Marcar como gerando
    await db.update(solicitacoes)
      .set({ status: 'gerando', updated_at: new Date() })
      .where(eq(solicitacoes.id, solicitacaoId));
    
    try {
      // ... lógica existente de buscar variant, renderizar, upload ...
      const arquivoUrl = await renderAndUpload(/* ... */);
      
      // 2. Sucesso
      await db.update(solicitacoes)
        .set({ 
          status: 'concluido', 
          arquivo_url: arquivoUrl,
          erro_geracao: null,
          updated_at: new Date() 
        })
        .where(eq(solicitacoes.id, solicitacaoId));
      
      return { success: true, url: arquivoUrl };
    } catch (error) {
      logger.error({ solicitacaoId, error }, 'art generation failed');
      
      // 3. Erro
      await db.update(solicitacoes)
        .set({ 
          status: 'erro', 
          erro_geracao: error instanceof Error ? error.message : String(error),
          updated_at: new Date() 
        })
        .where(eq(solicitacoes.id, solicitacaoId));
      
      throw error;  // re-throw pra caller saber
    }
  }

Adaptar nomes de campos/funções conforme o que já existe no arquivo.

## Critério de validação

1. Submeter um form de automation (ex: form-cartao-comemorativo). 
   Imediatamente após, verificar no DB:
     SELECT status, erro_geracao FROM solicitacoes 
     ORDER BY id DESC LIMIT 1;
   
   Sequência esperada (consultando algumas vezes durante render):
   - 'recebido' (logo após submit)
   - 'gerando' (durante render)
   - 'concluido' (após render OK)

2. Forçar erro: cause uma falha (ex: temporariamente comente as 
   credenciais R2 no .env, restart). Submeter form. Após alguns 
   segundos:
     SELECT status, erro_geracao FROM solicitacoes 
     ORDER BY id DESC LIMIT 1;
   
   Esperado: status='erro', erro_geracao com mensagem clara.

3. UI: /solicitacao.html?id=X de uma solicitação com status='erro' 
   deve mostrar alguma indicação visual (badge vermelho ou similar). 
   Se não mostrar, o STATUS_SOLICITACAO da Tarefa 1 não foi aplicado 
   corretamente.
