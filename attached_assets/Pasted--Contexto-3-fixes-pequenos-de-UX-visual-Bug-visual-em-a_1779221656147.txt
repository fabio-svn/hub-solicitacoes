## Fix 1 — Bug do onerror em admin-assets.html

Em public/admin-assets.html, função renderGrid (~linha 257).

LOCALIZAR a linha:
```js
<img src="${esc(a.url)}" alt="${esc(a.filename)}" loading="lazy" onerror="this.parentElement.innerHTML='<div class=\\"no-thumb\\">🖼</div>'">
```

PROBLEMA: o `\\"` vira `\"` no HTML, que não é escape válido em atributo. 
O `"` fecha o onerror prematuramente e o resto vaza pro DOM, quebrando 
o layout do grid de assets.

SUBSTITUIR essa linha por:
```js
<img src="${esc(a.url)}" alt="${esc(a.filename)}" loading="lazy" onerror="window.handleAssetImgError(this)">
```

E ADICIONAR uma nova função no topo do `<script>` da página (logo após 
as variáveis globais como `currentPage`, `allAssets`, etc.):

```js
window.handleAssetImgError = function(img) {
  const parent = img.parentElement;
  if (parent) parent.innerHTML = '<div class="no-thumb">🖼</div>';
};
```

Essa abordagem evita problemas de escape — a lógica fica em JS puro,
fora do atributo HTML.

## Fix 2 — Logo do header redireciona pra /index.html

Em public/shell.js, função _buildHeader (~linha 51).

LOCALIZAR:
```js
<a href="/dashboard.html" class="app-header-logo">
  ${logoUrl ? `<img src="${logoUrl}" alt="SVN" height="22">` : `<span ...>SVN</span>`}
</a>
```

SUBSTITUIR o `href="/dashboard.html"` por `href="/index.html"`:
```js
<a href="/index.html" class="app-header-logo">
```

NÃO tocar nos outros links do shell (sidebar admin "Minhas solicitações" 
continua apontando pra /dashboard.html — correto).
