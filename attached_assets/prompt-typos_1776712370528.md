## 4. `form-apresentacoes.html` — Navegação incorreta ao validar arquivo base

```js
// Em submitForm(), localizar o bloco:
// DE:
if (!isAtualizar && tipoCriacao === 'base-existente' && !document.getElementById('arquivoBaseNova').files[0]) {
  alert('Envie o arquivo base para continuar.'); goStep(2); return;
}

// PARA:
if (!isAtualizar && tipoCriacao === 'base-existente' && !document.getElementById('arquivoBaseNova').files[0]) {
  goToStep2(); // reconfigurar o form corretamente antes de mostrar o erro
  setTimeout(() => {
    const fieldEl = document.getElementById('uploadBase');
    if (fieldEl) {
      fieldEl.classList.add('field-invalid');
      let errorEl = fieldEl.querySelector('.field-error');
      if (!errorEl) {
        errorEl = document.createElement('div');
        errorEl.className = 'field-error';
        fieldEl.appendChild(errorEl);
      }
      errorEl.textContent = 'Envie o arquivo base para continuar.';
      fieldEl.scrollIntoView({ behavior: 'smooth', block: 'center' });
    }
  }, 50);
  return;
}
```

## 5. `dashboard.html` — Bug latente no listener de clique fora dos dropdowns

```js
// Em initAdmin(), localizar:
// DE:
document.addEventListener('click', e => {
  if (!e.target.closest('#dropdownTipoWrap')) document.getElementById('dropdownTipo').style.display = 'none';
  if (!e.target.closest('#dropdownStatusWrap')) document.getElementById('dropdownStatus').style.display = 'none';
});

// PARA (adicionar guard para elementos removidos do DOM):
document.addEventListener('click', e => {
  if (!e.target || !document.contains(e.target)) return;
  if (!e.target.closest('#dropdownTipoWrap')) {
    const el = document.getElementById('dropdownTipo');
    if (el) el.style.display = 'none';
  }
  if (!e.target.closest('#dropdownStatusWrap')) {
    const el = document.getElementById('dropdownStatus');
    if (el) el.style.display = 'none';
  }
});
```

## OBSERVAÇÕES

- Nenhum desses arquivos precisa de build
- As correções 1 e 2 são de texto/lógica no form de assessores
- A correção 3 é cosmética mas evita problemas em ambientes mais restritivos
- A correção 4 garante que ao voltar do step 3 o form de apresentações
  reconfigurar corretamente os campos condicionais (título, uploadAtualizar, tipoCriacaoField)
- A correção 5 previne erros silenciosos no console quando elementos são
  removidos do DOM enquanto o listener ainda está ativo
