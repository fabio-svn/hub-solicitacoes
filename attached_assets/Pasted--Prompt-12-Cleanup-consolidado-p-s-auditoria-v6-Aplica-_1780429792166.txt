## H) Cache busting nos assets estáticos
═══════════════════════════════════════════════════════════════

Hoje os HTMLs referenciam `style.css`, `utils.js`, etc. sem versionamento. Quando você atualiza um deles, o browser pode servir versão em cache. Adicionar `?v=YYYYMMDD` força reload.

Rodar no terminal Replit (substitui todos os refs):
```bash
cd artifacts/api-server/public
DATE=$(date +%Y%m%d)
echo "Versão de cache: $DATE"

# CSS
find . -maxdepth 1 -name "*.html" -exec sed -i "s|href=\"style\.css\"|href=\"style.css?v=${DATE}\"|g" {} +

# JS módulos compartilhados
for js in utils.js toast.js config.js auth.js shell.js; do
  find . -maxdepth 1 -name "*.html" -exec sed -i "s|src=\"${js}\"|src=\"${js}?v=${DATE}\"|g" {} +
done

echo "✓ Cache busting aplicado em todos os HTMLs"
```

⚠️ A cada deploy importante que mude CSS/JS compartilhados, atualizar o `?v=` rodando o comando novamente (ou criar um script `update-cache.sh`).
