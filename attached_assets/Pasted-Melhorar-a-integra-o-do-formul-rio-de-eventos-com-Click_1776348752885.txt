# Regra obrigatória de mapeamento

Não assumir nomes cegamente.

Antes de preencher os campos, inspecionar o payload real do formulário de eventos e usar as chaves reais existentes.

Se o nome técnico no backend for diferente, adaptar corretamente.

## Tipos de valor

* Campos texto → string
* Número de convidados → número real
* URLs → enviar URL válida
* Sim / Não → string legível se esse for o formato já validado

---

# Robustez

Se algum campo não existir no payload:

* não quebrar integração
* pular campo
* logar ausência de valor

Se um campo falhar:

* continuar os demais

# Regras obrigatórias dos materiais

## Detectar automaticamente:

* checkboxes selecionados
* cards selecionados
* array de materiais
* estrutura atual usada no formulário

## Mostrar apenas:

* campos preenchidos
* valores válidos
* respostas relevantes

## Não mostrar:

* vazio
* null
* undefined
* placeholders

# Humanização obrigatória
