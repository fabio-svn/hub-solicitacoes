Melhorar o layout do card **Dados da Solicitação** para lidar com textos longos sem quebrar a estrutura de colunas.

## Problema atual

Algumas solicitações possuem campos com muito texto (ex: canais, títulos longos, conteúdo resumido).

Quando isso acontece:

* as colunas deixam de existir
* o grid se deforma
* campos empurram outros elementos
* o aproveitamento horizontal piora

## Objetivo

Manter layout em colunas, porém controlar o conteúdo dentro de cada célula.

---

# PARTE 1 — Quebra inteligente de texto

Todo valor textual dentro das células deve respeitar uma largura máxima interna.

## Aplicar nos valores:

* título
* canais
* setor
* público-alvo
* finalidade
* observações curtas
* etc.

## Regras CSS desejadas

* quebrar em nova linha quando atingir limite do container
* manter palavras inteiras sempre que possível
* não hifenizar
* não expandir a coluna

## Comportamento esperado

Exemplo:

Antes:
E Mail, Feed do Instagram, Linkedin, Whatsapp

Depois:
E Mail, Feed do Instagram,
Linkedin, Whatsapp

---

# PARTE 2 — Sem quebra feia de palavra

## Não usar:

* break-all agressivo
* hifenização automática

## Quero:

* quebra natural entre palavras
* leitura limpa
* linhas bem distribuídas

---

# PARTE 3 — Grid inteligente por tipo de campo

## Campos curtos (2 colunas)

Manter lado a lado:

* Solicitante
* Setor
* Prazo
* Público-alvo
* Unidade
* Tipo

## Campos médios

Permitir quebra interna mantendo coluna:

* Título
* Canais
* Finalidade curta

## Campos longos

Ocupar largura total automaticamente:

* Conteúdo
* Briefing
* Observações
* Descrição extensa

---

# PARTE 4 — Responsividade

## Desktop

Manter grid elegante com colunas.

## Tablet

Reduzir para 2 colunas confortáveis.

## Mobile

1 coluna.

---

# PARTE 5 — Visual

Preservar:

* boa hierarquia tipográfica
* espaçamento vertical consistente
* labels discretas
* valores fortes

---

# PARTE 6 — Entrega esperada

Informar:

1. como foi resolvida a quebra de texto
2. quais campos viram full width automaticamente
3. como ficou o grid desktop/mobile
4. arquivos alterados

---

# Prioridade máxima

Quero manter o layout sofisticado em colunas, mesmo quando os textos forem grandes.

Não quero voltar para lista vertical simples.
