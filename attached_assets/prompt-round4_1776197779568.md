## 1. THANK YOU PAGE — Resumo da solicitação

Após o check animado e o texto de confirmação, adicionar um card de resumo da solicitação enviada, com visual inspirado em confirmação de pedido e-commerce.

### 1a. Passar dados da solicitação para a thank you page

No `submitForm()` de **todos os formulários**, antes de redirecionar para `thankyou.html`, salvar o resumo em `sessionStorage`:

```js
// Adicionar antes do window.location.href = '/thankyou.html':
const resumo = {
  tipo: TIPO_SOLICITACAO_LABELS[formData.get('tipo_solicitacao')] || formData.get('tipo_solicitacao'),
  subtipo: formData.get('subtipo') || '',
  titulo: dados.nomeEvento || dados.tituloEvento || dados.titulo || dados.nomeCompleto || '',
  solicitante: dados.nome || Auth.getUserName(),
  setor: dados.setor || '',
  data: new Date().toLocaleDateString('pt-BR'),
  hora: new Date().toLocaleTimeString('pt-BR', { hour: '2-digit', minute: '2-digit' }),
  materiais: dados.materiais || [],
  natureza: dados.natureza || '',
  maturidade: dados.maturidade || '',
};
try { sessionStorage.setItem('svn_ultimo_resumo', JSON.stringify(resumo)); } catch {}
```

### 1b. Exibir card de resumo na thank you page

No `thankyou.html`, após o elemento `#ctaBtn` e antes do `#logoWrap`, adicionar o card de resumo:

```html
<div id="resumoCard" style="opacity:0;margin-top:28px;text-align:left;width:100%">
  <div style="border-top:1px solid rgba(255,248,243,0.12);padding-top:24px;margin-bottom:16px">
    <p style="font-size:0.7rem;font-weight:600;letter-spacing:0.08em;text-transform:uppercase;opacity:0.4;margin-bottom:12px;color:var(--paper-white)">Resumo da solicitação</p>
    <div id="resumoItens"></div>
  </div>
</div>
```

No `<script>` do `thankyou.html`, adicionar após as animações existentes:

```js
// Carregar e exibir resumo
try {
  const raw = sessionStorage.getItem('svn_ultimo_resumo');
  if (raw) {
    const resumo = JSON.parse(raw);
    const itens = [];

    if (resumo.tipo)        itens.push({ label: 'Tipo',        valor: resumo.tipo });
    if (resumo.titulo)      itens.push({ label: 'Identificação', valor: resumo.titulo });
    if (resumo.natureza)    itens.push({ label: 'Natureza',    valor: resumo.natureza === 'presencial' ? 'Presencial' : 'Online' });
    if (resumo.solicitante) itens.push({ label: 'Solicitante', valor: resumo.solicitante });
    if (resumo.setor)       itens.push({ label: 'Setor',       valor: resumo.setor });
    if (resumo.data)        itens.push({ label: 'Enviado em',  valor: resumo.data + ' às ' + resumo.hora });
    if (resumo.materiais && resumo.materiais.length > 0) {
      itens.push({ label: 'Itens solicitados', valor: resumo.materiais.length + ' item(ns)' });
    }

    if (itens.length > 0) {
      document.getElementById('resumoItens').innerHTML = itens.map(item => `
        <div style="display:flex;justify-content:space-between;align-items:baseline;padding:7px 0;border-bottom:1px solid rgba(255,248,243,0.07);font-size:0.82rem">
          <span style="color:rgba(255,248,243,0.45);font-weight:400">${item.label}</span>
          <span style="color:var(--paper-white);font-weight:600;text-align:right;max-width:60%">${item.valor}</span>
        </div>
      `).join('');

      // Animar entrada do card
      setTimeout(() => {
        const card = document.getElementById('resumoCard');
        card.style.transition = 'opacity 0.4s ease';
        card.style.opacity = '1';
      }, 1200);

      // Limpar sessionStorage após exibir
      sessionStorage.removeItem('svn_ultimo_resumo');
    }
  }
} catch {}
```

### 2c. Renderizar o fluxo no `openDrawer()`

Na função `openDrawer()` do `dashboard.html`, após definir `statusObj`, adicionar:

```js
// Renderizar fluxo de etapas
const tipoBase = item.tipo_solicitacao?.split('-')[0] === 'eventos' ? 'eventos' : '_default';
const fluxo = (typeof FLUXOS_ETAPAS !== 'undefined' && FLUXOS_ETAPAS[item.tipo_solicitacao])
  ? FLUXOS_ETAPAS[item.tipo_solicitacao]
  : (FLUXOS_ETAPAS[tipoBase] || FLUXOS_ETAPAS['_default']);

const statusAtual = item.status;
const etapasVisiveis = fluxo.filter(e => e.visivel);
const etapaExcepcional = fluxo.find(e => !e.visivel && e.id === statusAtual);

const idxAtual = etapasVisiveis.findIndex(e => e.id === statusAtual);

const fluxoEl = document.getElementById('drawerFluxo');
if (fluxoEl && fluxo.length > 0) {
  // Se status é excepcional (reprovado/em-espera), mostrar banner
  if (etapaExcepcional) {
    fluxoEl.innerHTML = `
      <div style="background:rgba(159,63,55,0.08);border:1px solid rgba(159,63,55,0.2);border-radius:8px;padding:10px 14px;font-size:0.85rem;font-weight:600;color:var(--ruby-red);display:flex;align-items:center;gap:8px">
        <svg width="14" height="14" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2"><circle cx="12" cy="12" r="10"/><line x1="12" y1="8" x2="12" y2="12"/><line x1="12" y1="16" x2="12.01" y2="16"/></svg>
        ${etapaExcepcional.label}
      </div>`;
  } else {
    // Fluxo normal — linha de progresso estilo e-commerce
    fluxoEl.innerHTML = `
      <div style="margin-bottom:8px">
        <p style="font-size:0.7rem;font-weight:600;letter-spacing:0.07em;text-transform:uppercase;opacity:0.4;margin-bottom:12px">Andamento da solicitação</p>
        <div style="display:flex;align-items:flex-start;gap:0;overflow-x:auto;padding-bottom:4px">
          ${etapasVisiveis.map((etapa, idx) => {
            const isConcluida = idx < idxAtual;
            const isAtual = idx === idxAtual;
            const isFutura = idx > idxAtual;
            const isUltima = idx === etapasVisiveis.length - 1;

            const corCirculo = isConcluida ? 'var(--sage-green)' : isAtual ? 'var(--ruby-red)' : 'rgba(34,27,25,0.15)';
            const corTexto = isAtual ? 'var(--carbon-black)' : isConcluida ? 'var(--sage-green)' : 'rgba(34,27,25,0.35)';
            const pesoTexto = isAtual ? '700' : '400';

            const iconeCirculo = isConcluida
              ? `<svg width="10" height="10" viewBox="0 0 24 24" fill="none" stroke="white" stroke-width="3"><polyline points="20 6 9 17 4 12"/></svg>`
              : isAtual
                ? `<div style="width:6px;height:6px;border-radius:50%;background:white"></div>`
                : '';

            return `
              <div style="display:flex;flex-direction:column;align-items:center;flex:1;min-width:60px;position:relative">
                ${!isUltima ? `<div style="position:absolute;top:10px;left:50%;width:100%;height:2px;background:${isConcluida ? 'var(--sage-green)' : 'rgba(34,27,25,0.1)'}"></div>` : ''}
                <div style="width:20px;height:20px;border-radius:50%;background:${corCirculo};display:flex;align-items:center;justify-content:center;flex-shrink:0;position:relative;z-index:1;transition:0.2s">
                  ${iconeCirculo}
                </div>
                <span style="font-size:0.65rem;font-weight:${pesoTexto};color:${corTexto};text-align:center;margin-top:6px;line-height:1.3;max-width:64px">${etapa.label}</span>
              </div>
            `;
          }).join('')}
        </div>
      </div>`;
  }
}
```

### ⚠️ CUIDADOS IMPORTANTES ao fazer os replaces:

1. **NÃO alterar** nomes de variáveis, ids, classes CSS, chaves de objetos JS, valores de `data-value`, `data-filter`, `name` de inputs, `id` de elementos HTML
2. **NÃO alterar** URLs, paths de arquivos, nomes de funções JS
3. **NÃO alterar** valores dentro de `const`, `let`, `var` que são identificadores (ex: `const URL_FORM_CRIACAO_PDF` não vira `const URL_FORM_CRIAÇÃO_PDF`)
4. **ALTERAR** apenas textos visíveis: conteúdo de `textContent`, `innerHTML`, `placeholder`, `label`, strings em `alert()`, strings em `textContent =`, atributos `title`, `data-tooltip`, `data-label`, conteúdo HTML entre tags
5. Para palavras ambíguas como "nao", "ja", "so", "ate", "esta", "sao" — fazer substituição **manual e contextual**, não global, para evitar quebrar variáveis

## OBSERVAÇÕES FINAIS

- O fluxo de etapas para tipos além de "eventos" será definido posteriormente — o `_default` serve como placeholder
- Os novos status (`alinhamentos`, `cotacao-aprovacao`, `aguardando-pagamento`, `aguardando-finalizacao`, `reprovado`, `em-espera`) precisam ser criados no ClickUp e mapeados no `CLICKUP_STATUS_MAP` do `config.js` quando as integrações forem definidas
- O card de resumo na thank you page usa `sessionStorage` — se o usuário recarregar a página, o resumo não aparece mais (comportamento intencional)
