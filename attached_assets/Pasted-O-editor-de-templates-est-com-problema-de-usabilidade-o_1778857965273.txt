O editor de templates está com problema de usabilidade: o modo "Editar" só mostra caixas vazias, o "Preview" só mostra o resultado sem boxes editáveis. Usuário não consegue ver o que está editando enquanto edita. Refatorar para um modo unificado tipo Figma:
1. Remover separação Editar/Preview

Eliminar o toggle de modo da toolbar.
O canvas sempre mostra a arte renderizada como base, com as caixas de layer sobrepostas pra edição.
Boxes sempre clicáveis e arrastáveis em cima do preview.

2. Visual das boxes (estado-dependente)

Não-selecionada, sem hover: outline tracejado bem discreto, sem label.

css  border: 1px dashed rgba(59,130,246,0.25);
  background: transparent;

Hover: outline mais visível, label aparece.

css  border: 1.5px dashed rgba(59,130,246,0.6);
Label visível no canto superior esquerdo.

Selecionada: outline sólido azul + label sempre visível.

css  border: 2px solid #3b82f6;
  box-shadow: 0 0 0 2px rgba(59,130,246,0.25);
Assim a arte renderizada fica respirando e visível, e as boxes só "aparecem" quando o usuário precisa delas.
3. Render imediato ao carregar template
Hoje doLivePreview() só dispara em markDirty(). Adicionar chamada explícita no final do loadTemplate(), depois de buscar config + sample-data, pra renderizar a arte imediatamente sem precisar de interação.
4. Render automático em mais eventos
Além dos drag/resize/property já cobertos, adicionar scheduleLivePreview() em:

Cada keystroke nos inputs do painel "Dados de teste" (debounce 1000ms)
Mudança no select de bg variant (debounce 300ms)
Adicionar/deletar layer

5. Feedback de rendering
Spinner pequeno (não-bloqueante) no canto da toolbar enquanto request está em voo. UI continua interativa — usuário pode arrastar a próxima caixa enquanto a anterior renderiza, e o AbortController cancela o request antigo.
6. Live toggle
Manter o switch "Live" na toolbar, mas com default ligado. Quando desligado, volta o botão "Atualizar preview" pra modo manual (caso de uso: usuário quer fazer várias mudanças seguidas sem renderizar a cada uma, depois aciona manual).
7. Botão Salvar permanece
Auto-render ≠ auto-save. Preview muda em tempo real, mas só persiste no DB quando clicar Salvar. Indicador "● não salvo" continua mostrando quando há mudanças pendentes.
Resultado esperado: ao abrir um template, você vê a arte renderizada completa com texto, imagens, tudo. Os layers ficam como contornos sutis. Clica numa layer → ela ganha destaque + propriedades aparecem na direita. Arrasta → ~700ms depois a arte re-renderiza com a nova posição. Mesma fluidez do Figma/Photoshop/Bannerbear.
