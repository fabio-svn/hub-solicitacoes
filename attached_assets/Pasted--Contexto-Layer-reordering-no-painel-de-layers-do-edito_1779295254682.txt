## Contexto

Layer reordering no painel de layers do editor não reflete corretamente 
no preview gerado pelo backend. O frontend usa convenção "índice 0 = 
frente" (z-index alto), mas o backend faz composite na ordem natural 
do array (último = frente). Resultado: ordem visual no preview é o 
INVERSO da ordem no painel.

Fix: backend itera o array em ordem reversa, alinhando com a convenção 
do frontend (índice 0 do array = layer mais à frente).

Arquivo: artifacts/api-server/src/services/template-renderer.ts

---

## Fix — Inverter ordem de composite das layers

LOCALIZAR o loop que processa as layers (~linhas 408-413):
```ts
for (const layer of template.layers) {
  if (layer.type === 'text-line')  await renderTextLine(layer, dataStr, composites, renderCtx);
  if (layer.type === 'text-block') await renderTextBlock(layer, dataStr, composites, renderCtx);
  if (layer.type === 'image')      await renderImage(layer, dataStr, composites, canvasW, canvasH);
  if (layer.type === 'shape')      await renderShape(layer, composites, canvasW, canvasH);
}
```

(A linha do shape pode variar — pode estar com outros parâmetros ou em 
posição diferente do snippet acima.)

SUBSTITUIR a linha `for (const layer of template.layers) {` por:

```ts
// Ordem do composite: começa pelo último (que fica no fundo) e termina 
// pelo primeiro (que fica na frente). Convenção: índice 0 do array = 
// layer mais frente, alinhada com z-index do frontend.
for (const layer of [...template.layers].reverse()) {
```

O resto do loop permanece idêntico.

## Critério de validação

1. Editor → criar template com 2 layers: text-block "Texto" + shape "Botão"
2. No painel de layers, deixar Texto no topo (índice 0) e Botão embaixo
3. Posicionar visualmente: Texto em cima da área do Botão (sobreposição)
4. Salvar e visualizar preview
5. Validar: Texto aparece NA FRENTE do Botão (não escondido atrás)
6. Reorganizar no painel: Botão no topo, Texto embaixo
7. Re-renderizar preview
8. Validar: agora Botão aparece NA FRENTE do Texto

Templates existentes podem aparecer "diferentes" após o fix — é esperado, 
porque a renderização agora está alinhada com a ordem do painel. Se algum 
template estava intencionalmente com layers "ao contrário pra funcionar", 
o usuário precisará reorganizá-las no editor.

## Não faça

- Não tocar no frontend (admin-templates.html) — a convenção dele está 
  correta
- Não tocar nas funções de render individuais (renderTextLine, renderImage, 
  renderShape, renderTextBlock)
- Não mudar o order do array em si — apenas a iteração
- Se outros geradores (pdf-renderer.ts) também iteram layers, AVALIAR se 
  precisam do mesmo fix (provavelmente sim — checar)
