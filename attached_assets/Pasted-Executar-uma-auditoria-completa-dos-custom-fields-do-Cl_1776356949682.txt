Executar uma **auditoria completa dos custom fields do ClickUp** usados pelo Hub, corrigindo incompatibilidades de tipo, mapeamento e campos que não estão sendo preenchidos corretamente.

## Contexto atualizado

Os campos de horário são diferentes e possuem IDs distintos:

* Horário descrito → `44c91638-ccb6-41fa-8f05-dcab3085f313`
* Horário → `196f6d96-7ca9-4e88-bbf5-598cbf375146`

Portanto, não há conflito de ID.
O problema precisa ser investigado como possível incompatibilidade de tipo ou mapeamento incorreto.

Também foi identificado que o campo:

* Número estimado de convidados → `d676846e-b66c-4c71-8173-7378d9db1f95`

não está sendo preenchido.

# Objetivo

Fazer revisão técnica completa dos custom fields usados nas integrações:

* Eventos
* Lista Geral

e garantir que cada campo seja enviado no formato correto para o ClickUp.

---

# Ações obrigatórias

## 1. Levantar tipo real de cada custom field no ClickUp

Para cada campo usado, identificar o tipo real:

* text
* textarea
* number
* date
* dropdown
* url
* checkbox
* labels
* outro

Não assumir pelo nome.

## 2. Verificar mapeamento real dos campos de horário

Validar o uso correto de:

* Horário descrito → `44c91638-ccb6-41fa-8f05-dcab3085f313`
* Horário → `196f6d96-7ca9-4e88-bbf5-598cbf375146`

Confirmar:

* qual deles deve receber string
* qual deles deve receber data/hora
* se ambos realmente fazem sentido no fluxo atual
* qual valor do payload está sendo usado para cada um

## 3. Validar formato enviado para cada tipo

### Exemplo:

Se campo = number
Não enviar `"123"`
Enviar `123`

### Se campo = date

Converter corretamente para o formato aceito pelo ClickUp

### Se campo = url

Garantir URL válida

### Se campo = text

Enviar string simples

## 4. Criar camada de normalização de valores

Antes de enviar custom field:

* detectar tipo
* converter valor automaticamente

Exemplos:

* `"123"` → `123`
* `"2026-05-01"` → valor convertido corretamente para campo de data
* `"11:20"` → string ou valor convertido conforme o tipo real do campo

## 5. Revisar especialmente estes campos prioritários

### Eventos

* Data do evento
* Horário
* Horário descrito
* Número estimado de convidados
* Custo estimado
* Rateio
* Canal
* Link de transmissão
* Todos os campos de palestrantes

### Lista Geral

* Prazo de entrega
* Arquivo principal
* Arquivo de apoio
* Dropdowns
* Tipo/Subtipo

## 6. Melhorar logs

Para cada custom field enviado, registrar:

* fieldId
* nome lógico
* tipo ClickUp
* valor original recebido
* valor convertido enviado
* sucesso ou erro da API

## 7. Robustez

Se um field falhar:

* não quebrar criação da task
* continuar os demais
* registrar erro detalhado
