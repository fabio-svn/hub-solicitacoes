- src/routes/webhook.ts (27 statuses)
- src/routes/clickup.ts (29 statuses)

Ação:
1. Criar artifacts/api-server/src/lib/clickup-status.ts:

   export const CLICKUP_STATUS_MAP: Record<string, string> = {

3. Se houver divergência entre os 2 maps em algum status, ESCOLHA O 
   VALOR DE clickup.ts e reporte os status divergentes.

## Item 2 — Logger pino consistente

Hoje template-renderer.ts usa console.*. Outros lugares usam logger pino.

Ação:
1. Buscar todos os console.*:
     grep -rn "console\." artifacts/api-server/src/

## Item 4 — Type safety Request via declaration merging

Hoje req.user, req.solicitacao etc são acessados sem tipo correto.

Ação:
1. Criar artifacts/api-server/src/types/express.d.ts:

   declare global {
     namespace Express {
       interface Request {
         user?: {
           id: number;
           email: string;
           role: 'admin' | 'gestor' | 'colaborador';
         };
         solicitacao?: any; // refinar conforme uso
       }
     }
   }
   export {};

2. Confirmar que tsconfig.json inclui esse arquivo (compilerOptions.types 
   ou include).

3. Remover `(req as any).user` casts existentes — usar req.user direto.

## Item 6 — Template renderer LRU cache + DEBUG_RENDER

Hoje assetCache é unbounded (cresce indefinidamente).

Ação:
1. Em src/services/template-renderer.ts, substituir assetCache 
   por uma LRU bounded. Pode usar `lru-cache` npm (já deve estar 
   no projeto — verificar) ou implementar Map com size limit.

   Tamanho sugerido: 100 entries.

2. Adicionar flag DEBUG_RENDER (env var). Quando ativa, logar:
   - Cada placeholder substituído
   - Cache hits/misses
   - Tempo total de render

   No código:
   const DEBUG = process.env.DEBUG_RENDER === '1';
   if (DEBUG) logger.info({ placeholder, value }, '...');

## Item 7 — UNIQUE constraint clickup_task_id

Antes de aplicar a migration, verificar se há duplicatas:

  SELECT clickup_task_id, COUNT(*) 
  FROM solicitacoes 
  WHERE clickup_task_id IS NOT NULL 
  GROUP BY clickup_task_id 
  HAVING COUNT(*) > 1;

Se retornar linhas, REPORTAR e NÃO aplicar — precisa resolver dupes 
antes. Se vazio, prosseguir.

Adicionar em src/index.ts DB_STATEMENTS (ou em migrations/):

  CREATE UNIQUE INDEX IF NOT EXISTS idx_solicitacoes_clickup_task_id 
    ON solicitacoes(clickup_task_id) 
    WHERE clickup_task_id IS NOT NULL;
