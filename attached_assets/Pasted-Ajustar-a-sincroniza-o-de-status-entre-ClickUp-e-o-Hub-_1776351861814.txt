Ajustar a sincronização de status entre ClickUp e o Hub para as **solicitações gerais**.

## Objetivo

Atualizar o Hub para refletir automaticamente os novos status das tarefas da Lista Geral do ClickUp, com as novas cores e nova ordem de processo.

A sincronização deve acontecer em tempo real sempre que o status da task mudar no ClickUp.

---

# NOVO FLUXO DE STATUS — SOLICITAÇÕES GERAIS

## Ordem do processo visível no hub

1. Recebido → cinza
2. Em análise → amarelo
3. Aguardando informação → marrom claro
4. Em produção → laranja
5. Em revisão → roxo
6. Em aprovação → azul
7. Concluído → verde

## Status invisível no fluxo, mas visível quando ativo

* Cancelado → vermelho

### Regra:

O status **Cancelado** não deve aparecer na régua/etapas padrão do processo.
Ele só deve aparecer quando a solicitação estiver efetivamente cancelada.

---

# REGRAS DE SINCRONIZAÇÃO

## 1. O Hub deve acompanhar o status da task no ClickUp

Se a task no ClickUp mudar de status, o status da solicitação correspondente no hub deve ser atualizado automaticamente.

## 2. Atualização em tempo real

Implementar atualização em tempo real usando o mecanismo mais robusto disponível para ClickUp.

### Preferência:

* usar webhook do ClickUp para eventos de atualização de task/status

### Se já existir polling:

* substituir ou complementar por webhook
* evitar polling desnecessário se webhook resolver

## 3. Persistência

Quando o status mudar no ClickUp, persistir a mudança no banco/local store do hub para que:

* dashboard
* lista de solicitações
* drawer
* timeline
* badges
* filtros

reflitam o novo status corretamente.

---

# MAPEAMENTO DE STATUS

Criar/ajustar o mapeamento de status da Lista Geral para este padrão interno do hub:

* Recebido → `recebido`
* Em análise → `em-analise`
* Aguardando informação → `aguardando`
* Em produção → `em-producao`
* Em revisão → `em-revisao`
* Em aprovação → `em-aprovacao`
* Concluído → `concluido`
* Cancelado → `cancelado`

## IMPORTANTE

Usar correspondência robusta:

* ignorar diferenças pequenas de caixa
* normalizar acentos se necessário
* evitar falha por string levemente diferente
