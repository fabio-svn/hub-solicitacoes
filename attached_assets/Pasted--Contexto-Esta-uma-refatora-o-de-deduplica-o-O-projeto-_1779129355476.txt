## Contexto

Esta é uma refatoração de deduplicação. O projeto tem várias listas e 
funções espalhadas em múltiplos arquivos. O objetivo é estabelecer uma 
fonte única de verdade pra cada um, sem mudar comportamento externo.

Cobre 8 itens (P1.1, P1.2, P1.3, P1.4, P1.11, P1.12, P1.16, P1.17).

## Estratégia geral

Para cada item: identifica todas as duplicações via grep, escolhe o 
arquivo canônico, refatora os consumidores pra usar a fonte canônica, 
deleta as duplicações.

## Tarefa 5 — DRAWER_FIELD_LABELS estendido com metadata

Hoje em solicitacao.html há duas Sets soltos:
  const WIDE_KEYS = new Set([...20 keys...]);
  const SKIP_KEYS = new Set(['idSolicitacao', 'natureza', 'setor', 'nome', 'materiaisDetalhes']);

E em config.js há DRAWER_FIELD_LABELS = { key: 'Label', ... }.

Ação:

1. Em config.js, transforme DRAWER_FIELD_LABELS em objetos com metadata:

     const DRAWER_FIELD_LABELS = {
       conteudo: { label: 'Conteúdo', wide: true },
       finalidade: { label: 'Finalidade', wide: true },
       idSolicitacao: { label: 'ID', skip: true },
       // ... etc
     };

2. Pra retrocompatibilidade, exporte também uma versão flat 
   (DRAWER_FIELD_LABELS_FLAT) que é { key: 'Label' }, caso outros 
   arquivos consumam só o label.

3. Em solicitacao.html, substitua WIDE_KEYS.has(key) por 
   DRAWER_FIELD_LABELS[key]?.wide. Substitua SKIP_KEYS.has(key) por 
   DRAWER_FIELD_LABELS[key]?.skip.

4. Delete as Sets WIDE_KEYS e SKIP_KEYS de solicitacao.html.
